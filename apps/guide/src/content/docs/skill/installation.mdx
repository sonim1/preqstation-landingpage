`preqstation-skill` is the core agent-side package for PREQSTATION. It owns the `preqstation` worker skill, documents the lifecycle rules, and provides the Claude Code helper plugin plus Codex/Gemini worker install paths.

## Prerequisites

- Node.js 22+ installed
- A running PREQSTATION deployment at `https://<your-domain>`
- Access to the PREQSTATION web app if you want to use the recommended remote MCP login flow
- A PREQSTATION API token only if you plan to use shell-helper mode or the legacy local stdio bridge

## Choose a Worker Install

### Claude Code Plugin (Recommended)

For Claude Code, install the plugin. It includes the packaged `preqstation` worker skill and helper commands.

```bash
claude plugin marketplace add https://github.com/sonim1/preqstation-skill
claude plugin install preqstation@preqstation
```

Then open Claude Code and run:

```text
/preqstation:setup
```

`/preqstation:setup` can add or verify the user-scoped `preqstation` MCP connection, fetch projects when `preq_list_projects` is available, and save repo mappings in `~/.preqstation-dispatch/projects.json`.

### Worker-Only Installs

Use worker-only install when you do not want the Claude plugin helpers, or when installing for Codex or Gemini CLI:

```bash
npx skills add sonim1/preqstation-skill -g -a claude-code
npx skills add sonim1/preqstation-skill -g -a codex
npx skills add sonim1/preqstation-skill -g -a gemini-cli
```

PREQ engine values match these agent targets:

```bash
claude-code
codex
gemini-cli
```

## Dispatcher Paths Live Elsewhere

This page is only for **worker setup**.

If you also want queued dispatch from the web app, use [OpenClaw Installation](/guide/openclaw/setup) for the Telegram/OpenClaw dispatcher path.

## Recommended MCP Setup (HTTP + OAuth)

The recommended MCP runtime now lives in the PREQSTATION service itself at `/mcp`. Claude Code, Codex, and Gemini CLI should connect to that remote endpoint and complete the browser OAuth flow when prompted.

## Claude Code

Register the remote PREQSTATION MCP endpoint:

<Tabs>
<TabItem label="Install">
```bash
claude mcp add -s user --transport http preqstation https://<your-domain>/mcp
```
</TabItem>
<TabItem label="Update">
If already registered, remove and re-add:

```bash
claude mcp remove preqstation
```

Then run the install command above.
</TabItem>
<TabItem label="Verify">
```bash
claude mcp list
claude mcp get preqstation
```
</TabItem>
</Tabs>

## When Authentication Starts

- Codex often probes the server during `add`, so the browser login flow may start immediately.
- Gemini CLI may also prompt during `mcp add` or on the first real tool call, depending on the installed CLI build.
- Claude Code usually saves the server first and may show `Needs authentication` until first use.
- In Claude Code, authentication commonly starts when you first use the server or run `claude mcp get preqstation`.

## Worker vs Dispatcher

- **Worker setup** means the agent can call PREQ tools directly through MCP.
- **Dispatcher setup** means the web app can queue a task to another runtime.
- **Engine** still remains separate and can be `claude-code`, `codex`, or `gemini-cli`.

If you only want direct worker usage, this page is enough. If you want queued dispatch from the web app, continue with [OpenClaw Installation](/guide/openclaw/setup).

## Gemini CLI

Gemini CLI uses the same worker + remote MCP path when the local Gemini CLI supports remote HTTP MCP.

Register the remote PREQSTATION MCP endpoint:

<Tabs>
<TabItem label="Install">
```bash
gemini mcp add --scope user --transport http preqstation https://<your-domain>/mcp
```
</TabItem>
<TabItem label="Verify">
```bash
gemini mcp list
```
</TabItem>
<TabItem label="Run">
```bash
gemini
# or
gemini -p "Read and execute the PREQSTATION task instructions from this workspace."
```
</TabItem>
</Tabs>

If your installed Gemini CLI does not expose remote HTTP MCP, use shell-helper mode instead:

```bash
export PREQSTATION_API_URL="https://preqstation.example.com"
export PREQSTATION_TOKEN="preq_xxxxxxxxxxxxxxxxx"
export PREQSTATION_ENGINE="gemini-cli"  # optional fallback
```

## Shell Helper or Legacy Local Bridge (Optional)

If MCP is unavailable, you can still use the shell helper or the legacy local stdio bridge.

Set these variables first:

```bash
export PREQSTATION_API_URL="https://preqstation.example.com"
export PREQSTATION_TOKEN="preq_xxxxxxxxxxxxxxxxx"
export PREQSTATION_ENGINE="claude-code"  # or codex, gemini-cli
```

## JSON Configuration Example

If you must manually configure the legacy local stdio bridge in settings:

```json
{
  "mcpServers": {
    "preqstation": {
      "command": "node",
      "args": ["/ABSOLUTE/PATH/TO/preqstation-skill/scripts/preqstation-mcp-server.mjs"],
      "env": {
        "PREQSTATION_API_URL": "https://preqstation.example.com",
        "PREQSTATION_TOKEN": "preq_xxxxxxxxxxxxxxxxx"
      }
    }
  }
}
```

<Aside>
Replace `/ABSOLUTE/PATH/TO/preqstation-skill` with the actual installation path. Relative paths will not work. The recommended remote `/mcp` setup above does not need a local script path.
</Aside>

## Troubleshooting

**Remote MCP shows `Needs authentication`:**
- Confirm the server URL ends with `/mcp`
- Run `claude mcp get preqstation`, `codex mcp get preqstation`, or `gemini mcp list` to trigger or verify the first real connection
- Complete the browser login flow when prompted
- In Claude Code, prefer a single user-scoped `preqstation` MCP registration. Multiple local or project-scoped entries can cause repeated auth prompts.

**Legacy local bridge fails to start:**
- Verify Node.js is installed: `node --version`
- Check the script path is absolute
- Confirm `PREQSTATION_API_URL` is valid (should start with `https://`)

**Shell helper or legacy bridge returns `401 Unauthorized`:**
- Verify token format (should start with `preq_`)
- Test connectivity: `curl -H "Authorization: Bearer $PREQSTATION_TOKEN" "$PREQSTATION_API_URL/api/health"`

**Token rejected:**
- Generate a new token from the PREQSTATION web app `/api-keys` page
- Ensure token has not expired
