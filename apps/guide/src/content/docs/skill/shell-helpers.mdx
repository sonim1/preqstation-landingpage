When MCP is unavailable, source `scripts/preqstation-api.sh` to use shell wrapper functions. All mutation helpers accept an `engine` parameter to track which agent performed the action.

## Setup

<Tabs>
<TabItem label="Load Script">
```bash
source /path/to/preqstation-skill/scripts/preqstation-api.sh
```
</TabItem>
<TabItem label="With Env Vars">
```bash
export PREQSTATION_API_URL="https://your-domain.vercel.app"
export PREQSTATION_TOKEN="preq_xxxxx"
source /path/to/preqstation-skill/scripts/preqstation-api.sh
```
</TabItem>
</Tabs>

<Aside type="note">
Requires `jq` for JSON parsing. Install with `brew install jq` (macOS) or `apt-get install jq` (Linux).
</Aside>

## Read-Only Functions

These functions fetch data and don't require engine tracking.

### preq_get_tasks

List tasks with optional filters.

```bash
preq_get_tasks [status] [label]
```

**Parameters:**
- `status` (optional) — `todo`, `in_progress`, `review`, `done`, `blocked`
- `label` (optional) — Label name to filter by

**Example:**
```bash
preq_get_tasks todo
preq_get_tasks in_progress claude
```

### preq_get_task

Fetch single task details.

```bash
preq_get_task <task_id>
```

**Parameters:**
- `task_id` (required) — Task ticket (e.g., `PROJ-123`) or UUID

**Example:**
```bash
preq_get_task PROJ-284
```

### preq_get_project_settings

Fetch project deployment strategy.

```bash
preq_get_project_settings <project_key>
```

**Parameters:**
- `project_key` (required) — Project key (e.g., `PROJ`)

**Example:**
```bash
preq_get_project_settings PROJ
```

## Mutation Functions

All mutation functions accept an optional `engine` parameter: `claude`, `codex`, or `gemini`.

### preq_create_task

Create a new task.

```bash
preq_create_task '<json_payload>'
```

**Parameters:**
- JSON payload must include: `title`, `projectKey`, `engine`

**Example:**
```bash
preq_create_task '{"title":"Implement rate limiting","projectKey":"PROJ","engine":"claude"}'
```

### preq_start_task

Move task to in_progress.

```bash
preq_start_task <task_id> [engine]
```

**Example:**
```bash
preq_start_task PROJ-284 claude
```

### preq_update_task_status

Update task status.

```bash
preq_update_task_status <task_id> <status> [engine]
```

**Status values:** `inbox`, `todo`, `in_progress`, `in_review`, `done`, `archived`, `review` (alias for `in_review`)

**Example:**
```bash
preq_update_task_status PROJ-284 review claude
```

### preq_plan_task

Upload plan and move inbox → todo.

```bash
preq_plan_task <task_id> <plan_markdown> [engine]
```

**Example:**
```bash
PLAN="## Plan\n\n1. Add middleware\n2. Write tests"
preq_plan_task PROJ-284 "$PLAN" claude
```

### preq_complete_task

Upload result and move to review.

```bash
preq_complete_task <task_id> <summary> [engine] [pr_url] [tests] [notes] [branch_name]
```

**Parameters:**
- `task_id` — Task ticket or UUID
- `summary` — Brief completion summary (required)
- `engine` — Agent engine (optional)
- `pr_url` — Pull request URL (optional)
- `tests` — Test command or results (optional)
- `notes` — Additional notes (optional)
- `branch_name` — Git branch name (optional)

**Example:**
```bash
preq_complete_task PROJ-284 \
  "Implemented rate limiting for login endpoint" \
  claude \
  "https://github.com/org/repo/pull/123" \
  "npm run test" \
  "All checks passed" \
  "preqstation/PROJ/rate-limit"
```

### preq_review_task

Verify and move to done.

```bash
preq_review_task <task_id> [engine] [test_cmd] [build_cmd] [lint_cmd]
```

**Example:**
```bash
preq_review_task PROJ-284 claude "npm test" "npm run build" "npm run lint"
```

### preq_block_task

Mark task as blocked.

```bash
preq_block_task <task_id> <reason> [engine]
```

**Example:**
```bash
preq_block_task PROJ-284 "Missing required Redis environment variables" claude
```

### preq_delete_task

Permanently delete task.

```bash
preq_delete_task <task_id>
```

**Example:**
```bash
preq_delete_task PROJ-284
```

### preq_resolve_branch_name

Resolve canonical branch name for a task.

```bash
preq_resolve_branch_name <task_id> [fallback_branch]
```

**Example:**
```bash
preq_resolve_branch_name PROJ-284 "feature/rate-limit"
```

## Environment Variables

Ensure these are set before sourcing the script:

```bash
export PREQSTATION_API_URL="https://your-domain.vercel.app"
export PREQSTATION_TOKEN="preq_xxxxxxxxxxxxxxxxx"
```

## Error Handling

All functions return the raw API response. Parse errors with `jq`:

```bash
preq_get_task PROJ-284 | jq '.error' -r
```

Check HTTP status by examining the response. The script does not hide API errors.
