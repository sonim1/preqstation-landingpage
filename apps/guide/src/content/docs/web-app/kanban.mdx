## Task Statuses

| Status | Meaning | Action |
|---|---|---|
| **Inbox** | New tasks waiting for planning | Agent should run `preq_plan_task` to move to Todo |
| **Todo** | Planned and ready for agents | Agent can start work with `preq_start_task` |
| **Hold** | Blocked or paused | Agent or user resolves the blocker, then retries |
| **Ready** | Implementation submitted for verification | Choose a QA engine, queue a run, review history, then move to Done after verification |
| **Done** | Task verified and complete | Archive or reopen if needed |

Archived tasks are hidden from the active board and handled through manual archive actions.

## Ready Lane QA

The Ready lane includes QA runs for each project branch. Use the QA action on a Ready task to:

- choose the engine before queueing
- review the run history for that project and branch
- see statuses as `queued`, `running`, `passed`, or `failed`
- keep task workflow state separate from QA run state

Telegram must be configured before queueing QA runs, because the queue action sends the OpenClaw command through Telegram.

## Creating a Task

Click **+ New Task** to create a task:

1. **Title** ‚Äî What needs to be done (concise)
2. **Description** ‚Äî Context and detailed requirements
3. **Engine** ‚Äî Which agent will execute: `claude-code`, `codex`, or `gemini-cli`
4. **Priority** ‚Äî `highest`, `high`, `medium`, `low`, `lowest`
5. **Labels** ‚Äî Custom tags for organization (e.g., `backend`, `urgent`)
6. **Acceptance Criteria** ‚Äî Checklist of what "done" means

## Drag and Drop

Drag task cards between columns to update status:

- **Inbox ‚Üí Todo** ‚Äî Task is planned and ready
- **Todo ‚Üî Hold** ‚Äî Pause or resume blocked work
- **Todo/Hold ‚Üí Ready** ‚Äî Implementation is complete and ready for verification
- **Ready ‚Üí Done** ‚Äî Verification passed
- **Done ‚Üí Todo/Hold** ‚Äî Reopen for follow-up work

## Today Focus

Pin important tasks to the **Today** section:

1. Open a task card
2. Click the **Pin** icon
3. Pinned tasks appear at the top of the board

Use this to surface your current priorities.

## Command Palette

Press `/` or `Cmd+K` to open the command palette:

- Search tasks by title or ID
- Quick-navigate to a task
- Filter by status or label

## Task Card Details

Each card shows:

- **Task ID** ‚Äî Unique identifier (e.g., `PROJ-123`)
- **Title** ‚Äî Task name
- **Priority** ‚Äî Color-coded indicator
- **Engine** ‚Äî Which agent will work on it
- **Status indicator** ‚Äî Current workflow state with timestamp
- **Execution overlay** ‚Äî Optional `Queued` or `Working` badge independent from the column
- **QA run history** ‚Äî On Ready cards, a modal shows queued/running/passed/failed runs for the branch
- **Labels** ‚Äî Custom tags

Click a card to view full details, edit, or add comments.

## Bulk Actions

Select multiple tasks:

1. Hold `Cmd/Ctrl` and click cards
2. Use the action menu to:
   - Change priority
   - Add/remove labels
   - Archive completed tasks
   - Reassign engine
