## Prerequisites

Before you begin, ensure you have:

- **Node.js 22** or later
- **PostgreSQL 13+** (self-hosted or managed)
- **GitHub access** to the current private PREQSTATION web app development repository

## Environment Variables

Create a `.env.local` file in the project root. Copy from `.env.example` and fill in the required values:

| Variable           | Required | Description                                                                  |
| ------------------ | -------- | ---------------------------------------------------------------------------- |
| `AUTH_SECRET`      | Yes      | HMAC session signing secret. Generate: `openssl rand -base64 48`              |
| `OWNER_EMAIL`      | Yes      | The single owner account email                                               |
| `OWNER_PASSWORD`   | Yes      | Owner login password (min 8 chars)                                           |
| `DATABASE_URL`     | Yes      | PostgreSQL connection string                                                 |
| `ALLOWED_ORIGINS`  | No       | Comma-separated CORS origins (e.g., `http://localhost:3000`)                 |
| `PORT`             | No       | HTTP port (default: 3000)                                                    |

<Aside>
`drizzle-kit` reads `DATABASE_URL` from `.env.local`, so use `.env.local` for local setup before running database commands.
</Aside>

<Aside type="note">
Any standard PostgreSQL provider that exposes a compatible connection string works with PREQSTATION, including managed services and self-hosted deployments.
</Aside>

## Installation Steps

### 1. Clone and Install

The public PREQSTATION repository is [github.com/sonim1/preqstation](https://github.com/sonim1/preqstation). Until the web app source is moved there, local setup still uses the current private development repository:

```bash
git clone git@github.com:sonim1/project-manager.git projects-manager
cd projects-manager
npm install
```

If you do not have access yet, ask the maintainer to invite your GitHub account before continuing.

### 2. Configure Environment

```bash
cp .env.example .env.local
# Edit .env.local with your PostgreSQL connection string and owner credentials
```

### 3. Database Migration

```bash
npm run db:migrate
```

This applies the checked-in Drizzle migrations to your database.

<Aside type="tip">
If you later move to a new PostgreSQL provider, follow the [database migration guide](/guide/web-app/database-migration).
</Aside>

### 4. Start Development Server

```bash
npm run dev
```

The app opens at `http://localhost:3000`. Unauthenticated visits to `/` redirect to `/login`.

## First Login

1. Navigate to `http://localhost:3000/login`
2. Use the email and password from your `.env.local` file
3. You are now logged in as the owner

<Aside>
PREQSTATION is single-owner. All data is scoped to this account at the database level via PostgreSQL RLS.
</Aside>

## Configure Telegram (Optional)

To enable Telegram notifications:

1. Go to **Settings > Telegram**
2. Enter your Telegram bot token
3. Test the connection
4. Token is encrypted at rest with AES-GCM

## Configure API Keys

Issue tokens for agents to authenticate:

1. Open `/api-keys` after signing in, or click **API Keys** in the workspace sidebar
2. Use the **Create API Key** form to enter a name and optional expiration
3. Click **Create Key**
4. Copy the token from the success panel right away. It starts with `preq_` and is shown only once.

Use these tokens in your agent environment or `preqstation-skill` MCP configuration:

```bash
export PREQSTATION_API_URL=http://localhost:3000
export PREQSTATION_TOKEN=preq_your_new_token
```
