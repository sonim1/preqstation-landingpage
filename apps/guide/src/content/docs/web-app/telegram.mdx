Telegram is optional for general task work, but it is required for Ready-lane QA run queueing. The current documented setup uses it for explicit `Send to Telegram` actions, the OpenClaw dispatcher path, and the QA queue command rather than as a full automatic notification bus for every task event.

## Dispatcher Choice in the UI

The current guide documents the Telegram/OpenClaw dispatch action:

- **Send to Telegram** -> queues `dispatch_target=telegram`

Ready-lane QA queueing also goes through Telegram, so QA runs cannot be queued until Telegram is configured.

The dispatcher target is separate from the task `engine`. The OpenClaw path can still launch `claude-code`, `codex`, or `gemini-cli` if that runtime is installed on the dispatcher machine.

## Setup

### 1. Create a Telegram bot

1. Message [@BotFather](https://t.me/BotFather)
2. Run `/newbot`
3. Complete bot creation
4. Copy the bot token

### 2. Configure it in PREQSTATION

1. Open **Settings > Telegram**
2. Paste the bot token
3. Add your chat ID
4. Click **Test Connection**

### 3. Confirm the chat ID

The test message reveals the chat ID that PREQSTATION will use for outgoing messages and relays.

## Token Security

Telegram bot tokens are encrypted at rest in `user_settings` using AES-GCM with a key derived from `AUTH_SECRET`.

Stored format:

```text
v1.{base64urlIV}.{base64urlCiphertext}
```

## What Telegram Is Used For

Telegram is best treated as a delivery channel for:

- explicit task messages sent from the UI through **Send to Telegram**
- copied Telegram/OpenClaw commands
- OpenClaw relay messages

Use Telegram when you want the OpenClaw dispatcher path or Ready-lane QA queueing.

- For the Telegram/OpenClaw path, continue with [OpenClaw Installation](/guide/openclaw/setup)

### Disable notifications

1. Open **Settings > Telegram**
2. Turn off **Enable notifications**

## Relaying Requests to OpenClaw

Telegram can also carry OpenClaw requests. Prefer natural language or an explicit skill call rather than the old fixed `CLAUDE PROJ-123` format.

Examples:

```text
Use Codex to implement PROJ-123 in the example project.
Start PROJ-123 in sparse mode using Claude Code.
!/skill preqstation-dispatch review PROJ-123 using gemini-cli
```

The OpenClaw runtime receives the message, resolves the target project, creates a worktree, and launches the requested agent.

<Aside>
If your Telegram bot token is exposed, revoke it in [@BotFather](https://t.me/BotFather) and save a new token in PREQSTATION.
</Aside>
