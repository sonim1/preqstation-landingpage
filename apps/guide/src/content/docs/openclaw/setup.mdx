`preqstation-dispatcher` is the durable public dispatcher package for PREQSTATION. OpenClaw loads it as a plugin, then the plugin resolves projects, creates worktrees, writes `.preqstation-prompt.txt`, and launches Claude Code, Codex, or Gemini CLI as a detached process.

Some package, plugin, and config identifiers may still mention `preqstation-openclaw` for compatibility. Treat that as a legacy technical identifier unless a command explicitly requires it.

Use this page when you want the **OpenClaw dispatcher** path.

Use OpenClaw when you want:

- Telegram-triggered dispatch through the OpenClaw channel
- copied OpenClaw dispatch commands
- OpenClaw-native `/preqsetup` project mapping
- detached, worktree-first execution on an OpenClaw host

## Before You Install

You need:

- OpenClaw installed on the dispatcher host
- local checkouts for the projects PREQSTATION will dispatch
- at least one worker CLI available on the OpenClaw host: `claude`, `codex`, or `gemini`
- the matching PREQ worker setup from [Worker Installation](/guide/skill/installation)
- Telegram settings only if the web app should send dispatch messages through Telegram

If you only want an agent to work directly from its own terminal, stop at worker setup. OpenClaw is for queued dispatch from the web app or an OpenClaw chat.

## Install the Plugin

Install from npm on the machine where OpenClaw runs:

```bash
openclaw plugins install @sonim1/preqstation-dispatcher --dangerously-force-unsafe-install
openclaw gateway restart
```

If the standalone dispatcher CLI is already installed, it can run the plugin install command:

```bash
preqstation-dispatcher install openclaw
openclaw gateway restart
```

For active local development, link a checkout:

```bash
openclaw plugins install --link --dangerously-force-unsafe-install /path/to/preqstation-dispatcher
openclaw gateway restart
```

The plugin intentionally uses local process execution to create git worktrees and launch detached coding CLIs, so current OpenClaw installs require `--dangerously-force-unsafe-install`.

## Verify Installation

```bash
openclaw plugins inspect preqstation-dispatcher
openclaw status --all
```

## Project Path Mapping

After install, map PREQ project keys to local checkouts on the OpenClaw host.

<Tabs>
<TabItem label="Auto">
```text
/preqsetup auto PROJ=https://github.com/example/project
```

`/preqsetup auto` scans local git repos under `PREQSTATION_REPO_ROOTS` when set, otherwise under `~/projects`, matches local git `origin` URLs against the provided repo URLs, and stores successful matches in OpenClaw plugin config.
</TabItem>
<TabItem label="Manual">
```text
/preqsetup set PROJ /absolute/path/to/project
/preqsetup status
```
</TabItem>
<TabItem label="Shared Store">
```text
/preqsetup import
```

Use this when another dispatcher host already populated `~/.preqstation-dispatch/projects.json`.
</TabItem>
</Tabs>

Useful setup commands:

```text
/preqsetup
/preqsetup auto
/preqsetup import
/preqsetup set <PROJECT_KEY> <ABSOLUTE_PATH>
/preqsetup status
/preqsetup unset PROJ
```

Resolution order:

1. Absolute path mentioned directly in the message
2. Saved OpenClaw plugin mapping
3. Shared `~/.preqstation-dispatch/projects.json`
4. Legacy `MEMORY.md` fallback

## Worktree Root

OpenClaw uses `OPENCLAW_WORKTREE_ROOT` for task worktrees.

```bash
export OPENCLAW_WORKTREE_ROOT="/tmp/openclaw-worktrees"
```

Default: `/tmp/openclaw-worktrees`

Each task runs inside a dedicated worktree:

```text
<OPENCLAW_WORKTREE_ROOT>/<project_key>/<branch_slug>/
```

Detached runtime artifacts are written inside that worktree:

```text
.preqstation-dispatch/<engine>.pid
.preqstation-dispatch/<engine>.log
```

## Engine Keys

OpenClaw resolves PREQ engine keys to local CLI binaries:

| Engine key | Local binary |
|---|---|
| `claude-code` | `claude` |
| `codex` | `codex` |
| `gemini-cli` | `gemini` |

Default engine key: `claude-code`

Make sure the selected CLI is installed in a stable executable location that the OpenClaw gateway can inherit.

## Dispatch Commands

The OpenClaw adapter intercepts explicit PREQ dispatch messages before the normal chat run:

```text
!/skill preqstation-dispatch implement PROJ-284 using codex branch_name="task/proj-284-example"
/skill preqstation-dispatch review PROJ-284 using claude-code
/skill preqstation-dispatch ask PROJ-285 using gemini-cli ask_hint="Acceptance criteria"
```

Telegram/OpenClaw sends from the web app use the same OpenClaw command shape.

## Next Steps

1. Install or update the plugin
2. Restart OpenClaw
3. Map project keys with `/preqsetup auto` or `/preqsetup set`
4. Verify `claude`, `codex`, or `gemini` is available to the gateway process
5. Continue to [Execution](/guide/openclaw/execution)

<Aside>
If OpenClaw cannot resolve a project path from the message, plugin config, shared mapping file, or legacy fallback, add an explicit mapping before dispatching.
</Aside>
