`preqstation-dispatcher` is the durable public dispatcher repo for OpenClaw runtimes. It does not ship Telegram or webhook infrastructure by itself; the runtime imports this repo to resolve projects, create worktrees, and launch coding agents.

Some package, plugin, and config identifiers still use `preqstation-openclaw` for compatibility. Treat that as a technical legacy identifier, not the preferred public repo name.

Use this page when you want the **OpenClaw dispatcher** path.

Use OpenClaw when you want:

- Telegram-triggered dispatch
- copied Telegram/OpenClaw commands
- OpenClaw project-key and path resolution
- worktree-first execution managed by the OpenClaw runtime

## ClawHub Import

Repository URL:

```text
https://github.com/sonim1/preqstation-dispatcher
```

<Tabs>
<TabItem label="ClawHub Web UI">

4. Confirm that `SKILL.md` is detected from the repo root
</TabItem>
<TabItem label="Manual Setup">

2. Keep `SKILL.md` at repository root
3. Point your OpenClaw runtime to that checkout

## Project Path Mapping

Real project mappings can come from an explicit path in the dispatch message, saved plugin config, the shared `~/.preqstation-dispatch/projects.json` file, or fallback `MEMORY.md`.

`/Users/kendrick/projects/preqstation-dispatcher/MEMORY.md` is only a sample fallback schema reference.

Use short lowercase keys that map to absolute checkout paths, for example:

```markdown
| key | cwd | note |
|-----|-----|------|
| proj | /Users/kendrick/projects/my-app | Main product repo |
| docs | /Users/kendrick/projects/preqstation-lp | Guide site |
```

Resolution order:

1. Absolute path mentioned directly in the message
2. Saved plugin mapping
3. Shared `~/.preqstation-dispatch/projects.json`
4. Fallback `MEMORY.md`

## Environment Variable

### OPENCLAW_WORKTREE_ROOT

Controls where task worktrees are created.

```bash
export OPENCLAW_WORKTREE_ROOT="/tmp/openclaw-worktrees"
```

Default: `/tmp/openclaw-worktrees`

## Worktree Layout

Each task runs inside a dedicated worktree:

```text
<OPENCLAW_WORKTREE_ROOT>/<project_key>/<branch_slug>/
```

Branch naming priority:

1. `branch_name="..."` from the message
2. Fallback `preqstation/<project_key>`

If a custom branch name does not already include the project key, OpenClaw normalizes it to:

```text
preqstation/<project_key>/<branch_name>
```

Example:

```text
/tmp/openclaw-worktrees/proj/preqstation-proj-rate-limit/
```

## Engine Keys

OpenClaw resolves engine identity with PREQ engine keys while launching local CLI binaries:

| Engine key | Local binary |
|---|---|
| `claude-code` | `claude` |
| `codex` | `codex` |
| `gemini-cli` | `gemini` |

Default engine key: `claude-code`

This means the OpenClaw path can still launch:

- Claude Code
- Codex
- Gemini CLI

Natural-language examples:

- `Start PROJ-284 in the example project using Claude Code.`
- `Use Codex to fix README command examples in the example project.`
- `Use Gemini CLI to draft notes for DOC-12 in the example project.`

## Next Steps

1. Import the dispatcher into your OpenClaw runtime
2. Confirm project key → path mappings exist in plugin config, the shared mapping file, or fallback memory
3. Set `OPENCLAW_WORKTREE_ROOT` if you do not want the default
4. Verify `claude`, `codex`, or `gemini` is installed locally
5. Continue to [Execution](/guide/openclaw/execution)

<Aside>
If OpenClaw cannot resolve a project path from the message, plugin config, shared mapping file, or fallback memory, add an explicit mapping before continuing.
</Aside>
