## Recommended Setup Path

For most users, PREQSTATION setup now starts in the hosted web app and uses the remote MCP endpoint at `/mcp`.

1. Sign in to your PREQSTATION deployment.
2. Open `/connections` and complete the provider setup you need for your worker or dispatcher path.
3. Register the remote MCP endpoint:

```bash
# Claude Code
claude mcp add --transport http preqstation https://<your-domain>/mcp

# Codex
codex mcp add preqstation --url https://<your-domain>/mcp
```

4. Complete the browser OAuth flow when prompted. Codex often starts this during `add`, while Claude Code commonly starts it on the first real request or `claude mcp get preqstation`.
5. Continue with the worker and optional dispatcher setup docs below.

## Continue with the Setup Paths

- **Web App Setup** — if you are still finishing hosted setup, self-hosted local development, or compatibility details, use [Web App Setup](/guide/web-app/setup)
- **Worker Connection** — if you just finished the web app path and have the access you need, continue now with [Worker Installation](/guide/skill/installation)
- **Dispatcher Setup** — add queued delivery only if you need dispatch from the web app: [Claude Code Dispatch Installation](/guide/claude-dispatch/installation) or [OpenClaw Installation](/guide/openclaw/setup)

Claude Code Dispatch and OpenClaw are alternative dispatcher paths. If you do not need queued dispatch from the web app, stop after worker setup.

## Compatibility: Local Web App Development

Use this branch only if you are self-hosting the web app, validating migration-era setup, or need a local development environment.

### Local Development Prerequisites

- **Node.js 22** — [Download](https://nodejs.org)
- **PostgreSQL** — Any managed or self-hosted PostgreSQL works, for example Supabase, Neon, Railway, or your own Postgres instance
- **GitHub access** — The public canonical repository is `sonim1/preqstation`, but local web app setup still requires access to the current private development repository

### 1. Clone the Repository

The public PREQSTATION repository is [github.com/sonim1/preqstation](https://github.com/sonim1/preqstation). Until the web app source is moved there, local setup still uses the current private development repository:

```bash
git clone git@github.com:sonim1/project-manager.git projects-manager
cd projects-manager
```

If you do not have access yet, ask the maintainer to invite your GitHub account before continuing.

### 2. Configure Environment

Copy `.env.example` to `.env.local` and fill in the required variables:

```bash
cp .env.example .env.local
```

Then edit `.env.local`:

```bash
# Required
AUTH_SECRET=<generate with: openssl rand -base64 48>
DATABASE_URL=postgresql://...  # Your PostgreSQL connection string

# Optional
ALLOWED_ORIGINS=https://your-app.vercel.app,https://your-app-git-main-user.vercel.app
CRON_SECRET=<optional cleanup route secret>
```

Any standard PostgreSQL provider that exposes a compatible connection string works with PREQSTATION.

### 3. Install Dependencies

```bash
npm install
```

### 4. Apply Database Migrations

```bash
npm run db:migrate
```

### 5. Start the Development Server

```bash
npm run dev
```

The app runs at `http://localhost:3000` by default. Set `PORT=` in `.env.local` before starting if you want a different local port. If you open `/` while signed out, PREQSTATION redirects you to `/login`. Fresh databases show the owner setup form so you can create the first account in the browser. Existing instances use the owner email and password stored in the `users` table.

## Compatibility: API Tokens and the Legacy Local Bridge

You only need `/api-keys` and `PREQSTATION_TOKEN` when you are using shell-helper mode or the legacy local MCP bridge. The recommended remote `/mcp` + OAuth setup above does not require issuing a token first.

1. Log in, then open `http://localhost:3000/api-keys`. You can also reach the same page from the workspace sidebar by clicking **API Keys**.
2. In the **Create API Key** form, keep the default expiration or choose one from **Expires In**.
3. Enter a name such as `local-dev` or `codex-laptop`, then click **Create Key**.
4. Copy the token from the green **New API key (shown once)** panel immediately. The value starts with `preq_` and the remaining characters use URL-safe letters, numbers, `_`, and `-`, for example `preq_abc123...`.
5. Save it into your local agent environment before you close the page, because PREQSTATION only shows the full token once.

Use the token with the app URL in your agent runtime only for those compatibility paths:

```bash
export PREQSTATION_API_URL=http://localhost:3000
export PREQSTATION_TOKEN=preq_your_new_token
```

For the full token-based compatibility details, see [API Keys](/guide/web-app/api-keys) and [Environment Variables](/guide/reference/environment-variables).
