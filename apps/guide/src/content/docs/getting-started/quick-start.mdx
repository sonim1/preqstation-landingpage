## Prerequisites

- **Node.js 22** — [Download](https://nodejs.org)
- **PostgreSQL** — Neon provides a free tier at [neon.tech](https://neon.tech)
- **GitHub access** — The public canonical repository is `sonim1/preqstation`, but local web app setup still requires access to the current private development repository

## Installation

### 1. Clone the Repository

The public PREQSTATION repository is [github.com/sonim1/preqstation](https://github.com/sonim1/preqstation). Until the web app source is moved there, local setup still uses the current private development repository:

```bash
git clone git@github.com:sonim1/project-manager.git projects-manager
cd projects-manager
```

If you do not have access yet, ask the maintainer to invite your GitHub account before continuing.

### 2. Configure Environment

Copy `.env.example` to `.env.local` and fill in the required variables:

```bash
cp .env.example .env.local
```

Then edit `.env.local`:

```bash
# Required
AUTH_SECRET=<generate with: openssl rand -base64 48>
DATABASE_URL=postgresql://...  # Your Neon connection string

# Optional
PORT=3000
ALLOWED_ORIGINS=http://localhost:3000
```

Owner credentials are not read from environment variables. After migrations, provision the owner row in PostgreSQL `users` instead. See [Web App Setup](/guide/web-app/setup) for the SQL steps.

### 3. Install Dependencies

```bash
npm install
```

### 4. Apply Database Migrations

```bash
npm run db:migrate
```

### 5. Provision the Owner User

Create or update the owner account in PostgreSQL after the migrations are applied. See [Web App Setup](/guide/web-app/setup) for the SQL example.

### 6. Start the Development Server

```bash
npm run dev
```

The app runs at `http://localhost:3000`. If you open `/` while signed out, PREQSTATION redirects you to `/login`. Sign in with the owner email and password stored in PostgreSQL.

## Connect an Agent

1. Log in, then open `/connections`.
2. Register the remote PREQSTATION `/mcp` endpoint in Claude Code, Codex, or Gemini CLI and complete the browser OAuth flow.
3. Review connected clients, last-used time, and expiry in `/connections`.
4. If you need shell-helper compatibility, `/api-keys` redirects to `/connections?legacy=1`.

Use bearer tokens only for shell-helper compatibility or direct REST automation:

```bash
export PREQSTATION_API_URL=http://localhost:3000
export PREQSTATION_TOKEN=preq_your_new_token
```

After that, continue with the setup for your agent client or MCP integration.

## Choose Your Path

Workers and dispatchers are separate in PREQSTATION.

1. Start with [Worker Installation](/guide/skill/installation)
2. Add [OpenClaw Installation](/guide/openclaw/setup) if you want `Send to Telegram`, copied Telegram/OpenClaw commands, or queued QA dispatch
3. If you do not need queued dispatch from the web app, stop after worker setup
