PREQSTATION is an execution platform for AI-agent-driven development workflows. It bridges developers and coding agents through a task queue, a web app, and agent-facing APIs.

## Core Systems and Dispatcher Paths

PREQSTATION separates the queued dispatch target from the task engine:

<CardGrid>
	<Card title="PREQSTATION Web App" icon="setting">
		Central web app. Manages tasks, renders the Kanban UI, exposes the REST API, and lets the user choose between `Send to Claude Code` and `Send to Telegram`.
	</Card>
	<Card title="preqstation-openclaw" icon="rocket">
		OpenClaw dispatcher path. OpenClaw imports it to resolve projects, create worktrees, and launch coding agents from Telegram or direct runtime messages.
	</Card>
	<Card title="preqstation-skill" icon="approve">
		Agent client. MCP access and shell helpers for workers, plus the local Claude dispatcher channel that consumes `dispatch_target=claude-code-channel`.
	</Card>
</CardGrid>

<Aside>
The public canonical repository is `sonim1/preqstation`. The web app source used for local development is still being migrated there, so setup instructions may reference the current development repository until that move is complete.
</Aside>

## Data Flow

### Task Creation and Notification

```text
User creates task in the Kanban UI
  -> PREQSTATION stores it in PostgreSQL
  -> user optionally chooses a dispatch target
  -> PREQ queues either Claude Code dispatcher or Telegram/OpenClaw relay
```

### Dispatcher Target vs Engine

- `dispatch_target=claude-code-channel` means the user chose **Send to Claude Code** in the web app.
- `dispatch_target=telegram` means the user chose **Send to Telegram** or copied a Telegram/OpenClaw command.
- `engine=claude-code | codex | gemini-cli` stays separate from the dispatcher target and decides which local CLI actually runs the task.

### Agent Trigger via OpenClaw or Claude Dispatch

```text
Telegram / OpenClaw path:
  developer chooses Send to Telegram or hands the copied command to OpenClaw
    -> OpenClaw resolves project + task + engine
    -> creates an isolated git worktree
    -> launches the requested coding agent

Claude Code dispatcher path:
  developer queues "Send to Claude Code" in the web app
    -> PREQ stores run_state=queued with dispatch_target=claude-code-channel
    -> local Claude dispatcher session emits the task into the dispatcher channel
    -> dispatcher launches the task engine recorded on the task
```

### Agent Execution and Status Reporting

<Mermaid
  chart={`

  A[Inbox] --> B[Todo]

```text
Agent lifecycle:
  inbox -> preq_plan_task -> todo
  todo or hold -> implement -> preq_complete_task -> ready
  ready -> verify -> preq_review_task -> done
  any blocked step -> preq_block_task -> hold
```

## Tech Stack

| Layer      | Technology                                  |
| ---------- | ------------------------------------------- |
| Framework  | Next.js 16 (App Router)                     |
| UI         | React 19, Mantine 8, Tabler Icons, Recharts |
| ORM        | Drizzle ORM                                 |
| Database   | Neon PostgreSQL (Row Level Security)        |
| Validation | Zod                                         |
| Auth       | Session cookies (HMAC-signed), bcryptjs     |

<Aside>
All data is owner-scoped via Neon PostgreSQL Row Level Security. PREQSTATION is designed for single-developer use.
</Aside>
