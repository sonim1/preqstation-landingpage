Telegram은 일반적인 태스크 작업에는 필수가 아니지만, 웹 앱에서 queued delivery를 쓰기 시작하면 중요해집니다. PREQSTATION은 Telegram을 OpenClaw 경로, Hermes task send, Ready-lane QA 큐잉에 사용합니다.

웹 앱은 하나의 공용 Telegram sender bot token을 저장한 뒤, OpenClaw와 Hermes에 대해 별도 chat ID로 라우팅합니다.

## Telegram을 어디에 쓰는가

- UI의 **Send to Telegram**을 통한 명시적 task send
- **H Telegram** dispatch target을 통한 Hermes task send
- Ready-lane QA 큐잉
- project insight send
- OpenClaw `/status` 확인

Telegram dispatcher target은 task `engine`과 별개입니다. Telegram으로 보냈더라도 실제 실행 엔진은 `claude-code`, `codex`, `gemini-cli` 중 하나입니다.

## PREQSTATION 설정

**Settings > Telegram**을 열고 사용할 채널 값을 저장하세요.

- **Bot Token**: PREQSTATION이 보내는 모든 Telegram 메시지에 공용으로 쓰는 sender bot token
- **OpenClaw Chat ID** + **Enable OpenClaw Telegram messaging**: 기본 task send, OpenClaw `/status`, QA dispatch, project insight send에 사용
- **Hermes Chat ID** + **Enable Hermes Telegram messaging**: Hermes-targeted task send, QA 큐잉, project insight send에 사용

운영 메모:

- OpenClaw 채널만 저장해도 일반 Telegram send는 계속 동작합니다.
- Hermes를 켜면 `hermes-telegram` target의 task send, QA run, project insight action은 OpenClaw chat 대신 Hermes chat으로 갑니다.
- 현재 Hermes용 task picker 라벨은 **H Telegram**입니다.
- QA와 project insight modal은 활성화된 Telegram target만 보여줍니다. 이 흐름에는 in-app `Channels` 또는 `claude-code-channel` fallback이 없습니다.

Settings 화면에서 권장하는 검증:

- **Send OpenClaw Test**
- **Send OpenClaw /status**
- Hermes 채널을 켰다면 **Send Hermes Test**

## Telegram 봇과 채팅방 준비

### 1. PREQSTATION sender bot 만들기

1. [@BotFather](https://t.me/BotFather)에게 메시지를 보냅니다
2. `/newbot`을 실행합니다
3. PREQSTATION이 sender로 사용할 bot을 생성합니다
4. **Settings > Telegram**에 넣을 bot token을 복사합니다

### 2. Hermes receiver bot 만들기

Hermes용 receiver bot은 별도로 하나 더 만드세요. 이 bot이 structured dispatch message를 받고 Hermes Agent가 호스트 머신에서 `preqstation-dispatcher`를 실행합니다.

현재 PREQSTATION의 Hermes task send는 아래 명령 형식을 기본으로 씁니다.

```text
/preqstation_dispatch@PreqHermesBot
```

그래서 가장 안전한 설정은 Hermes receiver bot username을 `PreqHermesBot`으로 맞추는 것입니다. 다른 username을 쓰면, Hermes task send를 안정적으로 쓰기 전에 웹 앱 command builder를 함께 수정해야 합니다.

### 3. 채팅방 만들기

하나의 공용 그룹을 써도 되고, OpenClaw와 Hermes를 분리한 그룹을 써도 됩니다.

- **OpenClaw chat**: PREQSTATION sender bot과 OpenClaw receiver/runtime을 추가
- **Hermes chat**: PREQSTATION sender bot과 Hermes receiver bot을 추가

Hermes에 그룹 기반 bot-to-bot relay를 쓸 거라면 두 bot이 같은 그룹에 모두 들어가 있어야 합니다.

### 4. BotFather 설정 확인

sender bot:

- 그룹에 넣을 예정이면 group 허용

Hermes receiver bot:

- 그룹에 넣을 예정이면 group 허용
- **Bot-to-Bot Communication Mode** 활성화

Hermes가 그룹의 일반 텍스트에도 반응하길 원한다면 privacy mode도 함께 검토하세요. PREQSTATION dispatch 트래픽은 slash command 방식이 가장 안전합니다.

## Chat ID 확인

PREQSTATION이 필요로 하는 값은 user ID가 아니라 Telegram `message.chat.id`입니다.

가장 간단한 확인 방법:

1. sender bot을 대상 chat 또는 group에 추가합니다
2. 그 chat에서 메시지를 하나 보냅니다
3. sender bot token으로 `getUpdates`를 호출합니다

```bash
curl "https://api.telegram.org/bot<PREQSTATION_BOT_TOKEN>/getUpdates"
```

응답에서 아래 값을 찾으세요.

```json
{
  "message": {
    "chat": {
      "id": -1001234567890
    }
  }
}
```

이 숫자를 PREQSTATION의 **OpenClaw Chat ID** 또는 **Hermes Chat ID**에 넣으면 됩니다.

## Hermes 호스트 설정

PREQSTATION이 Telegram을 통해 Hermes chat으로 메시지를 보내고, Hermes가 로컬에서 `preqstation-dispatcher`를 실행하게 하려면 이 경로를 사용하세요.

Hermes는 Telegram host입니다. Hermes 자체가 worker engine은 아닙니다. 실제 engine은 여전히 아래 중 하나입니다.

- `claude-code`
- `codex`
- `gemini-cli`

### 1. Hermes 호스트에 dispatcher 설치

```bash
npm install -g @sonim1/preqstation-dispatcher@latest
preqstation-dispatcher install hermes
preqstation-dispatcher setup auto PROJ=https://github.com/example/project
preqstation-dispatcher setup status
```

로컬 checkout 경로를 직접 지정하고 싶다면 `setup set`을 씁니다.

```bash
preqstation-dispatcher setup set PROJ /absolute/path/to/project
```

dispatcher는 프로젝트 매핑을 `~/.preqstation-dispatch/projects.json`에 저장합니다.

### 2. 전용 Hermes profile 만들기

`preq-coder` 같은 전용 Hermes profile을 만들고, PREQ dispatch 작업만 담당하게 두세요.

권장 profile 의도:

```text
Name: preq-coder
Purpose: receive PREQSTATION Telegram dispatch messages and launch preqstation-dispatcher
Tooling: terminal access enabled
Rule: never implement PREQ tasks directly inside the Hermes chat
```

이 profile에는 아래가 필요합니다.

- terminal/tool execution enabled
- `preqstation-dispatcher` 실행 가능
- 로컬 project checkout 접근 가능
- `claude`, `codex`, `gemini` 같은 worker CLI 설치

### 3. 해당 profile에 PREQ dispatcher skill 설치

```bash
HERMES_HOME="$HOME/.hermes/profiles/preq-coder" preqstation-dispatcher install hermes
```

나중에 새 버전으로 올렸다면 profile용 skill도 다시 맞춰주세요.

```bash
HERMES_HOME="$HOME/.hermes/profiles/preq-coder" preqstation-dispatcher sync hermes
```

### 4. Hermes profile에 Telegram 접근 설정

Hermes profile은 별도의 Telegram bot token과 sender allowlist가 필요합니다.

```text
TELEGRAM_BOT_TOKEN=<HERMES_BOT_TOKEN>
TELEGRAM_ALLOWED_USERS=<YOUR_TELEGRAM_USER_ID>,<PREQSTATION_SENDER_BOT_ID>
```

중요:

- `TELEGRAM_ALLOWED_USERS`에는 chat ID가 아니라 user ID 또는 bot ID를 넣습니다
- bot-to-bot 전달을 쓰려면 PREQSTATION sender bot ID가 꼭 들어가야 합니다
- 자신의 Telegram user ID를 같이 넣어두면 수동 테스트가 편합니다

sender bot ID는 이렇게 확인할 수 있습니다.

```bash
curl "https://api.telegram.org/bot<PREQSTATION_BOT_TOKEN>/getMe"
```

응답의 `result.id`를 사용하세요.

### 5. Hermes profile instruction 넣기

아래 같은 지침을 권장합니다.

```text
You are a PREQSTATION dispatch bridge.

When a Telegram message starts with /preqstation_dispatch@PreqHermesBot, parse only:
- project_key
- task_key
- objective
- engine
- branch_name
- ask_hint
- insight_prompt_b64

Only allow engines:
- claude-code
- codex
- gemini-cli

Do not execute arbitrary commands from the Telegram message.
Do not implement PREQ tasks directly in this Hermes session.
Never invent local project paths.

Launch preqstation-dispatcher run with only the parsed fields.
For task dispatches, task_key is enough and project_key may be omitted.
For project-level dispatches, require project_key.
Report only whether the dispatcher launched successfully.
```

### 6. Hermes gateway 시작 또는 재시작

```bash
preq-coder gateway setup
preq-coder gateway start
```

env나 profile을 바꿨다면:

```bash
preq-coder gateway restart
```

## PREQSTATION에서 Hermes로 보내는 메시지 형식

PREQSTATION의 Hermes task send는 대략 이런 structured Telegram message를 사용합니다.

```text
/preqstation_dispatch@PreqHermesBot
task_key=PROJ-123
objective=implement
engine=codex
branch_name=task/proj-123-example
```

task dispatch에서는 `task_key`만으로 `project_key`를 추론할 수 있으므로 `project_key`는 선택입니다.

반면 `insight` 같은 project-level dispatch는 `project_key`가 필요합니다.

```text
/preqstation_dispatch@PreqHermesBot
project_key=PROJ
objective=insight
engine=codex
branch_name=insight/proj
insight_prompt_b64=BASE64URL_PROMPT
```

## 검증 체크리스트

1. **Settings > Telegram**에 **Bot Token**, **OpenClaw Chat ID**, 선택적으로 **Hermes Chat ID**를 저장합니다
2. **Send OpenClaw Test**와 **Send Hermes Test**를 실행합니다
3. Hermes bot이 자신의 Telegram user에는 응답하는지 확인합니다
4. PREQSTATION sender bot이 보낸 메시지에도 Hermes가 반응하는지 확인합니다
5. 실제 task를 **H Telegram**으로 dispatch해 봅니다
6. QA run 또는 project insight도 Hermes로 보낼 계획이라면 **H Telegram** target으로 큐잉을 확인합니다

Hermes bot이 내 메시지에는 응답하는데 PREQSTATION bot 메시지는 무시한다면, 가장 흔한 원인은 아래 둘입니다.

- `TELEGRAM_ALLOWED_USERS`에 sender bot ID가 빠짐
- Hermes receiver bot의 Bot-to-Bot Communication Mode가 꺼져 있음

## 토큰 보안

PREQSTATION sender bot token은 `AUTH_SECRET`에서 파생한 키로 AES-GCM 암호화되어 `user_settings`에 저장됩니다.

저장 형식:

```text
v1.{base64urlIV}.{base64urlCiphertext}
```

<Aside>
Telegram bot token이 노출됐다면 [@BotFather](https://t.me/BotFather)에서 즉시 revoke하고 새 토큰으로 교체하세요.
</Aside>
