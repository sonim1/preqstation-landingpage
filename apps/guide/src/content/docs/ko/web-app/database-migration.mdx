`DATABASE_URL`로 연결할 수 있는 PostgreSQL이라면 PREQSTATION을 호스팅할 수 있습니다. 이 문서는 기존 설치를 한 PostgreSQL 제공자에서 다른 제공자로 옮길 때 사용하는 실무 중심 절차를 정리합니다.

<Aside type="caution">
점검 시간을 잡고 진행하세요. 이 가이드는 표준 PostgreSQL dump-and-restore 절차를 설명하며, 무중단 전환이나 제공자 간 복제를 보장하지 않습니다.
</Aside>

## 언제 마이그레이션하나요

- 데이터베이스 제공자를 바꿔야 할 때
- 배포 리전과 더 가까운 곳으로 데이터베이스를 옮기고 싶을 때
- 비용, 저장 용량, 성능 특성을 바꾸고 싶을 때

## 지원하는 데이터베이스 제공자

- `DATABASE_URL`로 접근할 수 있는 모든 PostgreSQL 제공자
- 예시: Supabase, Neon, Railway, self-hosted PostgreSQL

## 마이그레이션 절차

1. `pg_dump`로 현재 데이터베이스를 백업합니다.
2. 대상 PostgreSQL 데이터베이스를 만들고 연결이 되는지 확인합니다.
3. `pg_restore` 또는 `psql`로 백업을 복원합니다.
4. PREQSTATION 환경의 `DATABASE_URL`을 새 데이터베이스로 바꿉니다.
5. `npm run db:migrate`를 실행합니다.
6. PREQSTATION에 로그인해서 태스크 접근과 API 키 흐름을 확인합니다.

## 예시 명령

`pg_restore`를 사용할 계획이라면 custom format dump가 편합니다.

```bash
pg_dump \
  --format=custom \
  --no-owner \
  --no-privileges \
  --dbname="$SOURCE_DATABASE_URL" \
  --file=preqstation.dump

pg_restore \
  --no-owner \
  --no-privileges \
  --dbname="$TARGET_DATABASE_URL" \
  preqstation.dump
```

plain SQL dump를 선호한다면 `psql`로 복원할 수 있습니다.

```bash
pg_dump \
  --format=plain \
  --no-owner \
  --no-privileges \
  --dbname="$SOURCE_DATABASE_URL" \
  --file=preqstation.sql

psql "$TARGET_DATABASE_URL" < preqstation.sql
```

## 이전 후 확인할 일

- PREQSTATION이 실행되는 모든 곳의 `DATABASE_URL`을 새 값으로 업데이트합니다.
- 앱을 재시작하거나 다시 배포해 새 환경 변수를 반영합니다.
- 로그인해서 기존 태스크, API 키, owner 전용 흐름이 예상대로 동작하는지 확인합니다.
