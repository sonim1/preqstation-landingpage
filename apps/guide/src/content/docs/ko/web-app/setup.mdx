## 사전 준비

시작하기 전에 다음 항목을 준비하세요.

- **Node.js 22** 이상
- **PostgreSQL 13+** 이상
- 현재 private PREQSTATION 웹 앱 개발 저장소에 대한 **GitHub 접근 권한**

## 환경 변수

프로젝트 루트에 `.env.local` 파일을 만들고 `.env.example`을 복사한 뒤 필요한 값을 채우세요.

| 변수 | 필수 | 설명 |
| --- | --- | --- |
| `AUTH_SECRET` | 예 | HMAC 세션 서명 비밀값. 생성 예시: `openssl rand -base64 48` |
| `DATABASE_URL` | 예 | PostgreSQL 연결 문자열 |
| `ALLOWED_ORIGINS` | 아니오 | 쉼표로 구분한 CORS origin 목록, 예: `http://localhost:3000` |
| `PORT` | 아니오 | HTTP 포트, 기본값 `3000` |
| `CRON_SECRET` | 아니오 | cron 라우트용 선택 비밀값 |

<Aside>
`drizzle-kit`은 `.env.local`의 `DATABASE_URL`을 읽습니다. 로컬 DB 명령을 실행하기 전에 `.env.local`을 준비하세요.
</Aside>

owner 계정은 환경 변수로 넣지 않습니다. 마이그레이션 후 PostgreSQL `users` 테이블에 `email`, `password_hash`, `is_owner`를 채워 넣으세요.

## 설치 단계

### 1. 저장소 클론 및 설치

공개 PREQSTATION 저장소는 [github.com/sonim1/preqstation](https://github.com/sonim1/preqstation)입니다. 웹 앱 소스가 아직 그쪽으로 완전히 옮겨지지 않아, 로컬 설정은 현재 private 개발 저장소를 사용합니다.

```bash
git clone git@github.com:sonim1/project-manager.git projects-manager
cd projects-manager
npm install
```

아직 접근 권한이 없다면 계속 진행하기 전에 maintainer에게 GitHub 초대를 요청하세요.

### 2. 환경 변수 설정

```bash
cp .env.example .env.local
# 데이터베이스 URL과 계정을 포함해 .env.local 값을 수정합니다
```

### 3. 데이터베이스 마이그레이션

```bash
npm run db:migrate
```

이 명령은 저장소에 포함된 Drizzle 마이그레이션을 현재 데이터베이스에 적용합니다.

### 4. owner 계정 등록

bcrypt 해시를 만든 뒤 owner 행을 업데이트하거나 새로 추가합니다.

```bash
node -e "const bcrypt = require('bcryptjs'); bcrypt.hash(process.argv[1], 10).then(console.log)" 'replace-with-strong-password'
```

기존 데이터가 이미 owner 행에 연결돼 있다면:

```sql
UPDATE users
SET email = 'you@example.com',
    password_hash = '$2b$10$replace_me',
    is_owner = true,
    updated_at = NOW()
WHERE is_owner = true;
```

새 데이터베이스처럼 owner 행이 없다면:

```sql
INSERT INTO users (email, password_hash, is_owner)
VALUES ('you@example.com', '$2b$10$replace_me', true);
```

기존 인스턴스를 옮기는 경우, 프로젝트와 태스크 연결이 유지되도록 owner 행의 `id`는 유지하세요.

### 5. 개발 서버 시작

```bash
npm run dev
```

앱은 `http://localhost:3000`에서 열립니다. 로그인하지 않은 상태에서 `/`를 방문하면 `/login`으로 이동합니다.

## 첫 로그인

1. `http://localhost:3000/login`으로 이동합니다.
2. PostgreSQL에 저장한 owner 이메일과 비밀번호로 로그인합니다.
3. owner 계정으로 로그인된 상태가 됩니다.

<Aside>
PREQSTATION은 single-owner 구조입니다. 모든 데이터는 PostgreSQL RLS를 통해 이 계정 기준으로 격리됩니다.
</Aside>

## 에이전트 연결

1. `/connections`를 엽니다.
2. Claude Code나 Codex에 PREQSTATION 원격 `/mcp` 엔드포인트를 등록합니다.
3. 브라우저 OAuth를 완료합니다.
4. 연결된 클라이언트와 revoke 액션은 같은 페이지에서 확인할 수 있습니다.

셸 헬퍼가 필요하면 `/api-keys`가 `/connections?legacy=1`으로 리다이렉트됩니다.

## Telegram 설정 (선택 사항)

Telegram 알림을 사용하려면 다음 순서로 설정하세요.

1. **Settings > Telegram**으로 이동합니다.
2. Telegram 봇 토큰을 입력합니다.
3. 연결 테스트를 실행합니다.
4. 토큰은 AES-GCM으로 암호화되어 저장됩니다.

## 레거시 API 키

셸 헬퍼 호환이나 직접 REST 자동화가 필요할 때만 사용하세요.

1. 셸 헬퍼나 직접 REST 자동화가 이미 레거시 bearer token에 의존할 때만 계속 사용하세요.
2. `/api-keys`는 `/connections?legacy=1`으로 리다이렉트되며, 이 경로가 호환성 전용임을 보여줍니다.
3. 새 설치는 `/mcp` OAuth와 `/connections`를 기본으로 사용하세요.

이 토큰은 에이전트 환경 변수에 사용합니다.

```bash
export PREQSTATION_API_URL=http://localhost:3000
export PREQSTATION_TOKEN=preq_your_new_token
```
