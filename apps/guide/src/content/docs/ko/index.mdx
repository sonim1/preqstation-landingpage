PREQSTATION은 Claude, Codex, Gemini를 실제 프로젝트에 연결하기 위한 실행 시스템입니다.

## 추천 첫 단계

처음 PREQSTATION을 설정한다면 [빠른 시작](/guide/ko/getting-started/quick-start)부터 시작하세요. 웹 앱 실행, 로그인, 권장 원격 MCP 경로까지 가장 짧은 경로로 안내합니다. 전체 로컬 체크리스트를 보고 싶다면 [웹 앱 설정](/guide/ko/web-app/setup)으로 이어가면 됩니다.

## 설정 경로 선택

### 1. 웹 앱 설정

- [빠른 시작](/guide/ko/getting-started/quick-start) — 로컬 앱 실행, 로그인, 첫 API 키 발급까지 가장 빠르게 진행
- [웹 앱 설정](/guide/ko/web-app/setup) — 환경 변수, 마이그레이션, 첫 로그인 흐름까지 전체 로컬 설정 확인

### 2. 워커 연결

- [워커 설치](/guide/ko/skill/installation) — 앱이 이미 실행 중일 때 Claude, Codex, Gemini worker를 연결

### 3. 디스패처 설정

웹 앱에서 queued dispatch가 필요할 때만 dispatcher를 추가하세요.

- [OpenClaw 설치](/guide/ko/openclaw/setup) — 웹 앱에서 Telegram/OpenClaw 전달 사용

## PREQSTATION 구성 이해하기

- **PREQSTATION 웹 앱** — 칸반 UI, REST API, dispatch 액션 UI, Telegram relay, 실행 오버레이를 제공하는 중심 웹 앱
- **preqstation-skill** — Claude Code, Codex, Gemini CLI가 태스크를 읽고 상태를 갱신할 때 쓰는 MCP/worker 패키지
- **preqstation-dispatcher** — Telegram이나 복사한 런타임 메시지를 받는 선택적 OpenClaw-native dispatcher 경로. 일부 package/plugin/config identifier는 호환성을 위해 아직 `preqstation-openclaw`를 사용합니다.

task의 **engine**이 실제로 `claude-code`, `codex`, `gemini-cli` 중 어떤 CLI를 띄울지 결정하고, queued dispatcher 경로는 그와 별도로 동작합니다.

처음 설치라면 웹 앱부터 시작하세요. 앱과 API 키가 이미 있으면 워커 연결로 넘어가고, 큐 기반 전달이 필요할 때만 dispatcher 경로를 추가하면 됩니다.

이 시스템은 “이슈를 모아두는 백로그”보다, 이미 범위가 정해진 작업을 실제로 실행시키는 데 맞습니다.

태스크를 만드세요. 에이전트가 계획하고 구현하고 검증합니다. 마지막 판단과 승인만 당신이 합니다.
