PREQSTATION은 AI 에이전트 기반 개발 워크플로우를 위한 실행 플랫폼입니다. 개발자와 코딩 에이전트를 태스크 큐, 웹 앱, 에이전트용 API로 연결합니다.

처음 설정 중이라면 이 페이지보다 [빠른 시작](/guide/ko/getting-started/quick-start)에서 로컬 실행과 첫 API 키 발급을 먼저 끝내는 편이 좋습니다. 전체 설치 항목을 하나씩 확인하려면 [웹 앱 설정](/guide/ko/web-app/setup)을 참고하세요.

## 핵심 시스템과 레거시 ingress

<CardGrid>
  <Card title="PREQSTATION 웹 앱" icon="setting">
    중심 웹 앱입니다. 태스크를 관리하고, 칸반 UI를 렌더링하고, REST API를 노출하고, Claude Code dispatch queue를 만들며, Telegram/OpenClaw relay도 계속 지원합니다.
  </Card>
  <Card title="preqstation-openclaw" icon="rocket">
    레거시 실행 ingress입니다. OpenClaw가 이 패키지를 불러 프로젝트를 찾고 worktree를 만들고 코딩 에이전트를 실행합니다.
  </Card>
  <Card title="preqstation-skill" icon="approve">
    에이전트 클라이언트입니다. 코딩 에이전트용 MCP/worker 패키지이자, 로컬 Claude plugin skeleton과 실험적 dispatch channel 파일을 함께 담습니다.
  </Card>
</CardGrid>

<Aside>
공개 canonical 저장소는 `sonim1/preqstation`입니다. 다만 웹 앱 소스는 아직 그쪽으로 완전히 옮겨지지 않아, 로컬 설정 문서에서는 당분간 현재 개발 저장소를 함께 안내합니다.
</Aside>

## 데이터 흐름

### 태스크 생성

```text
사용자가 칸반 UI에서 태스크를 생성
  -> PREQSTATION 웹 앱이 PostgreSQL에 저장
  -> 필요하면 Telegram / OpenClaw로 relay
  -> 또는 Claude Code direct dispatch queue 생성
```

### OpenClaw 또는 Claude dispatch를 통한 실행

```text
Telegram / OpenClaw 경로:
  개발자가 OpenClaw에 태스크 요청
    -> OpenClaw가 프로젝트 + 태스크 + 엔진 해석
    -> 격리된 git worktree 생성
    -> 요청된 코딩 에이전트 실행

Claude Code 경로:
  개발자가 웹 앱에서 Send to Claude Code 실행
    -> PREQ가 dispatch_target=claude-code-channel 과 함께 queued 저장
    -> 로컬 Claude dispatch channel 이 dispatcher session 으로 태스크를 emit
```

## 기술 스택

| 레이어 | 기술 |
| ---------- | ------------------------------------------- |
| Framework  | Next.js 16 (App Router)                     |
| UI         | React 19, Mantine 8, Tabler Icons, Recharts |
| ORM        | Drizzle ORM                                 |
| Database   | Neon PostgreSQL (Row Level Security)        |
| Validation | Zod                                         |
| Auth       | Session cookies (HMAC-signed), bcryptjs     |

<Aside>
모든 데이터는 Neon PostgreSQL RLS를 통해 단일 owner 기준으로 격리됩니다. 현재 PREQSTATION은 개인 개발자 사용 흐름에 맞춰 설계되어 있습니다.
</Aside>
