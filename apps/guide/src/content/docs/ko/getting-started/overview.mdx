PREQSTATION은 AI 에이전트 기반 개발 워크플로우를 위한 실행 플랫폼입니다. 개발자와 코딩 에이전트를 태스크 큐, 웹 앱, 에이전트용 API로 연결합니다.

시스템 다이어그램보다 먼저 제품 이야기부터 보고 싶다면 [철학](/guide/ko/getting-started/philosophy) 문서를 먼저 읽으세요.

## 추천 첫 단계

처음 PREQSTATION을 설정한다면 [빠른 시작](/guide/ko/getting-started/quick-start)부터 시작하세요. 웹 앱 실행, 로그인, 첫 API 키 발급까지 가장 짧은 경로로 안내합니다. 전체 로컬 체크리스트를 보고 싶다면 [웹 앱 설정](/guide/ko/web-app/setup)으로 이어가면 됩니다.

## 설정 경로 순서

- **웹 앱 설정** — 가장 빠른 첫 실행은 [빠른 시작](/guide/ko/getting-started/quick-start), 전체 로컬 체크리스트는 [웹 앱 설정](/guide/ko/web-app/setup)
- **워커 연결** — 앱이 이미 실행 중이고 API 키를 만든 뒤에는 [워커 설치](/guide/ko/skill/installation)로 이동
- **디스패처 설정** — 웹 앱에서 queued dispatch가 필요할 때만 추가: [Claude Code Dispatch 설치](/guide/ko/claude-dispatch/installation) 또는 [OpenClaw 설치](/guide/ko/openclaw/setup)

## 데이터 흐름

### 태스크 생성

```text
사용자가 칸반 UI에서 태스크를 생성
  -> PREQSTATION 웹 앱이 PostgreSQL에 저장
  -> 필요하면 dispatch target 선택
  -> Claude Code dispatcher 또는 Telegram/OpenClaw relay 큐 생성
```

### Dispatcher target과 engine의 차이

- `dispatch_target=claude-code-channel`은 웹 앱에서 **Send to Claude Code**를 고른 경우입니다.
- `dispatch_target=telegram`은 **Send to Telegram**을 누르거나 Telegram/OpenClaw용 명령을 복사한 경우입니다.
- `engine=claude-code | codex | gemini-cli`는 dispatcher와 별개이며, 실제로 어떤 CLI를 실행할지 정합니다.

### OpenClaw 또는 Claude dispatch를 통한 실행

```text
Telegram / OpenClaw 경로:
  개발자가 Send to Telegram을 누르거나 복사한 명령을 OpenClaw에 전달
    -> OpenClaw가 프로젝트 + 태스크 + 엔진 해석
    -> 격리된 git worktree 생성
    -> 요청된 코딩 에이전트 실행

Claude Code dispatcher 경로:
  개발자가 웹 앱에서 Send to Claude Code 실행
    -> PREQ가 dispatch_target=claude-code-channel 과 함께 queued 저장
    -> 로컬 Claude dispatcher session 이 태스크를 channel 로 emit
    -> dispatcher 가 task.engine 에 맞는 CLI를 실행
```

## 기술 스택

| 레이어 | 기술 |
| ---------- | ------------------------------------------- |
| Framework  | Next.js 16 (App Router)                     |
| UI         | React 19, Mantine 8, Tabler Icons, Recharts |
| ORM        | Drizzle ORM                                 |
| Database   | PostgreSQL (Row Level Security)             |
| Validation | Zod                                         |
| Auth       | Session cookies (HMAC-signed), bcryptjs     |

<Aside>
모든 데이터는 PostgreSQL RLS를 통해 단일 owner 기준으로 격리됩니다. 현재 PREQSTATION은 개인 개발자 사용 흐름에 맞춰 설계되어 있습니다.
</Aside>
