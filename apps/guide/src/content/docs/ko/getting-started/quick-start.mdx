## 권장 설정 경로

대부분의 사용자는 배포된 PREQSTATION 웹 앱에서 시작하고, `/mcp`의 원격 MCP 엔드포인트를 사용하는 것이 기본 경로입니다.

1. PREQSTATION 배포에 로그인합니다.
2. `/connections`를 열고 worker 또는 dispatcher 경로에 필요한 provider 설정을 완료합니다.
3. 원격 MCP 엔드포인트를 등록합니다.

```bash
# Claude Code
claude mcp add --transport http preqstation https://<your-domain>/mcp

# Codex
codex mcp add preqstation --url https://<your-domain>/mcp

# Gemini CLI
gemini mcp add --scope user --transport http preqstation https://<your-domain>/mcp
```

4. 브라우저 OAuth 흐름이 뜨면 완료합니다. Codex는 `add` 중에 바로 시작하는 경우가 많고, Claude Code는 첫 실제 요청이나 `claude mcp get preqstation`에서 시작되는 경우가 많습니다. Gemini CLI는 등록 시점이나 첫 실제 호출에서 시작될 수 있습니다.
5. 아래의 worker 및 선택적 dispatcher 설정 문서로 이어서 진행합니다.

## 같은 설정 경로로 이어가기

- **웹 앱 설정** — hosted 설정, self-hosted 로컬 개발, 호환성 세부 정보를 더 확인하려면 [웹 앱 설정](/guide/ko/web-app/setup)을 참고하세요
- **워커 연결** — 웹 앱 경로를 마쳤고 필요한 접근 준비가 끝났다면 이제 [워커 설치](/guide/ko/skill/installation)로 이어가면 됩니다
- **디스패처 설정** — 웹 앱에서 queued dispatch가 필요할 때만 추가하세요: [OpenClaw 설치](/guide/ko/openclaw/setup)

웹 앱의 queued dispatch가 필요 없다면 worker 설정까지만 하고 멈추면 됩니다.

## 호환성: 로컬 웹 앱 개발

이 경로는 웹 앱을 self-hosting해야 하거나, migration-era 설정을 검증해야 하거나, 로컬 개발 환경이 필요한 경우에만 사용하세요.

### 로컬 개발 사전 준비

- **Node.js 22** — [다운로드](https://nodejs.org)
- **PostgreSQL** — Supabase, Neon, Railway 같은 관리형 서비스나 직접 운영하는 PostgreSQL 모두 사용할 수 있습니다
- **GitHub 접근 권한** — 공개 기준 저장소는 `sonim1/preqstation`이지만, 로컬 웹 앱 설정에는 아직 현재 비공개 개발 저장소 접근 권한이 필요합니다

### 1. 저장소 클론

공개 PREQSTATION 저장소는 [github.com/sonim1/preqstation](https://github.com/sonim1/preqstation)입니다. 웹 앱 소스가 아직 그쪽으로 완전히 옮겨지지 않아, 로컬 설정은 현재 비공개 개발 저장소를 사용합니다.

```bash
git clone git@github.com:sonim1/project-manager.git projects-manager
cd projects-manager
```

아직 접근 권한이 없다면 계속 진행하기 전에 관리자에게 GitHub 초대를 요청하세요.

### 2. 환경 변수 설정

`.env.example`을 `.env.local`로 복사한 뒤 필요한 값을 채우세요.

```bash
cp .env.example .env.local
```

그다음 `.env.local`을 수정합니다.

```bash
# Required
AUTH_SECRET=<openssl rand -base64 48 로 생성>
DATABASE_URL=postgresql://...  # PostgreSQL 연결 문자열

# Optional
ALLOWED_ORIGINS=https://your-app.vercel.app,https://your-app-git-main-user.vercel.app
CRON_SECRET=<cleanup route용 선택 비밀값>
```

표준 PostgreSQL 연결 문자열을 제공하는 관리형 서비스나 직접 운영하는 PostgreSQL이면 PREQSTATION과 함께 사용할 수 있습니다.

### 3. 의존성 설치

```bash
npm install
```

### 4. 데이터베이스 마이그레이션

```bash
npm run db:migrate
```

### 5. 개발 서버 시작

```bash
npm run dev
```

앱은 기본적으로 `http://localhost:3000`에서 실행됩니다. 다른 로컬 포트를 쓰려면 시작 전에 `.env.local`에 `PORT=`를 추가하세요. 로그인하지 않은 상태에서 `/`를 열면 `/login`으로 리다이렉트됩니다. 새 데이터베이스라면 브라우저에서 첫 owner 계정을 만드는 설정 폼이 표시됩니다. 기존 인스턴스라면 `users` 테이블에 저장된 owner 이메일과 비밀번호로 로그인하세요.

## 호환성: API 토큰과 레거시 로컬 브리지

`/api-keys`와 `PREQSTATION_TOKEN`은 shell-helper 모드 또는 레거시 로컬 MCP 브리지를 사용할 때만 필요합니다. 위의 권장 원격 `/mcp` + OAuth 경로에서는 먼저 토큰을 발급할 필요가 없습니다.

1. 로그인한 뒤 `http://localhost:3000/api-keys`를 엽니다. 같은 화면은 워크스페이스 사이드바의 **API Keys**에서도 열 수 있습니다.
2. **Create API Key** 폼에서 만료 기간을 그대로 두거나 **Expires In**에서 원하는 값을 고릅니다.
3. 키 이름에 `local-dev` 또는 `codex-laptop`처럼 알아보기 쉬운 이름을 입력하고 **Create Key**를 클릭합니다.
4. 초록색 **New API key (shown once)** 패널에 표시된 토큰을 바로 복사합니다. 값은 `preq_`로 시작하고, 뒤에는 URL-safe 문자와 숫자, `_`, `-`가 이어집니다. 예: `preq_abc123...`
5. 이 페이지를 닫기 전에 토큰을 로컬 에이전트 환경 변수에 저장하세요. 전체 토큰은 한 번만 표시됩니다.

이 토큰은 해당 호환성 경로에서만 앱 URL과 함께 에이전트 런타임에 설정합니다.

```bash
export PREQSTATION_API_URL=http://localhost:3000
export PREQSTATION_TOKEN=preq_your_new_token
```

토큰 기반 호환성 설정 전체는 [API Keys](/guide/ko/web-app/api-keys)와 [Environment Variables](/guide/ko/reference/environment-variables)에서 확인하세요.
