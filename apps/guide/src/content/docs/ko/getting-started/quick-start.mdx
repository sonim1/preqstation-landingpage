## 사전 준비

- **Node.js 22** — [다운로드](https://nodejs.org)
- **PostgreSQL** — 클라우드 호스팅은 [neon.tech](https://neon.tech) 무료 플랜으로 시작할 수 있습니다
- **GitHub 접근 권한** — 공개 canonical 저장소는 `sonim1/preqstation`이지만, 로컬 웹 앱 설정은 아직 현재 private 개발 저장소 접근 권한이 필요합니다

## 설치

### 1. 저장소 클론

공개 PREQSTATION 저장소는 [github.com/sonim1/preqstation](https://github.com/sonim1/preqstation)입니다. 웹 앱 소스가 아직 그쪽으로 완전히 옮겨지지 않아, 로컬 설정은 현재 private 개발 저장소를 사용합니다.

```bash
git clone git@github.com:sonim1/project-manager.git projects-manager
cd projects-manager
```

아직 접근 권한이 없다면 계속 진행하기 전에 maintainer에게 GitHub 초대를 요청하세요.

### 2. 환경 변수 설정

`.env.example`을 `.env.local`로 복사한 뒤 필요한 값을 채우세요.

```bash
cp .env.example .env.local
```

그다음 `.env.local`을 수정합니다.

```bash
# Required
AUTH_SECRET=<openssl rand -base64 48 로 생성>
OWNER_EMAIL=your-email@example.com
OWNER_PASSWORD=<your-password>
DATABASE_URL=postgresql://...

# Optional
PORT=3000
ALLOWED_ORIGINS=http://localhost:3000
```

### 3. 의존성 설치

```bash
npm install
```

### 4. 데이터베이스 마이그레이션

```bash
npm run db:migrate
```

### 5. 개발 서버 시작

```bash
npm run dev
```

앱은 `http://localhost:3000`에서 실행됩니다. 로그인하지 않은 상태에서 `/`를 열면 `/login`으로 리다이렉트됩니다. `.env.local`에 넣은 이메일과 비밀번호로 로그인하세요.

## 첫 API 키 발급

1. 로그인한 뒤 `http://localhost:3000/api-keys`를 엽니다. 같은 화면은 워크스페이스 사이드바의 **API Keys**에서도 열 수 있습니다.
2. **Create API Key** 폼에서 만료 기간을 그대로 두거나 **Expires In**에서 원하는 값을 고릅니다.
3. 키 이름에 `local-dev` 또는 `codex-laptop`처럼 알아보기 쉬운 이름을 입력하고 **Create Key**를 클릭합니다.
4. 초록색 **New API key (shown once)** 패널에 표시된 토큰을 바로 복사합니다. 값은 `preq_`로 시작하고, 뒤에는 URL-safe 문자와 숫자, `_`, `-`가 이어집니다. 예: `preq_abc123...`
5. 이 페이지를 닫기 전에 토큰을 로컬 에이전트 환경 변수에 저장하세요. 전체 토큰은 한 번만 표시됩니다.

토큰과 앱 URL을 에이전트 런타임에 설정합니다.

```bash
export PREQSTATION_API_URL=http://localhost:3000
export PREQSTATION_TOKEN=preq_your_new_token
```

이제 `preqstation-skill` 또는 MCP 설정으로 넘어가면 됩니다.
