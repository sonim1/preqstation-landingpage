`preqstation-dispatcher`는 PREQSTATION의 durable public dispatcher package입니다. OpenClaw는 이 패키지를 plugin으로 로드하고, plugin은 프로젝트를 찾고 worktree를 만들고 `.preqstation-prompt.txt`를 쓴 다음 Claude Code, Codex, Gemini CLI를 detached process로 실행합니다.

일부 package/plugin/config identifier에는 호환성을 위해 아직 `preqstation-openclaw`가 남아 있을 수 있습니다. 명령이 명시적으로 요구하지 않는 한 legacy technical identifier로만 보세요.

이 페이지는 **OpenClaw dispatcher** 경로를 쓸 때 봅니다.

OpenClaw는 아래 경우에 적합합니다.

- OpenClaw 채널을 통한 Telegram-triggered dispatch
- 복사한 OpenClaw dispatch 명령 실행
- OpenClaw-native `/preqsetup` project mapping
- OpenClaw host에서 detached, worktree-first 실행

## Plugin 설치

OpenClaw가 실행되는 머신에서 npm package를 설치합니다.

```bash
openclaw plugins install @sonim1/preqstation-dispatcher --dangerously-force-unsafe-install
openclaw gateway restart
```

standalone dispatcher CLI가 이미 설치되어 있다면 CLI가 plugin 설치 명령을 대신 실행할 수 있습니다.

```bash
preqstation-dispatcher install openclaw
openclaw gateway restart
```

로컬 개발 중인 checkout을 연결하려면:

```bash
openclaw plugins install --link --dangerously-force-unsafe-install /path/to/preqstation-dispatcher
openclaw gateway restart
```

이 plugin은 git worktree를 만들고 detached coding CLI를 실행하기 위해 로컬 process 실행을 사용합니다. 그래서 현재 OpenClaw install에는 `--dangerously-force-unsafe-install`이 필요합니다.

## 설치 확인

```bash
openclaw plugins inspect preqstation-dispatcher
openclaw status --all
```

## 프로젝트 경로 매핑

설치 후 OpenClaw host에서 PREQ project key를 로컬 checkout에 매핑합니다.

<Tabs>
<TabItem label="Auto">
```text
/preqsetup auto PROJ=https://github.com/example/project
```

`/preqsetup auto`는 `PREQSTATION_REPO_ROOTS`가 있으면 그 아래를, 없으면 `~/projects` 아래의 git repo를 스캔합니다. 로컬 `origin` URL과 전달한 repo URL을 비교해서 맞는 checkout을 OpenClaw plugin config에 저장합니다.
</TabItem>
<TabItem label="Manual">
```text
/preqsetup set PROJ /absolute/path/to/project
/preqsetup status
```
</TabItem>
<TabItem label="Shared Store">
```text
/preqsetup import
```

다른 dispatcher host가 이미 `~/.preqstation-dispatch/projects.json`을 채워두었을 때 사용합니다.
</TabItem>
</Tabs>

유용한 setup 명령:

```text
/preqsetup
/preqsetup auto
/preqsetup import
/preqsetup set <PROJECT_KEY> <ABSOLUTE_PATH>
/preqsetup status
/preqsetup unset PROJ
```

해석 순서:

1. 메시지에 직접 포함된 절대 경로
2. 저장된 OpenClaw plugin mapping
3. 공유 `~/.preqstation-dispatch/projects.json`
4. legacy `MEMORY.md` fallback

## Worktree Root

OpenClaw는 task worktree에 `OPENCLAW_WORKTREE_ROOT`를 사용합니다.

```bash
export OPENCLAW_WORKTREE_ROOT="/tmp/openclaw-worktrees"
```

기본값: `/tmp/openclaw-worktrees`

각 task는 전용 worktree에서 실행됩니다.

```text
<OPENCLAW_WORKTREE_ROOT>/<project_key>/<branch_slug>/
```

Detached runtime artifact는 그 worktree 안에 기록됩니다.

```text
.preqstation-dispatch/<engine>.pid
.preqstation-dispatch/<engine>.log
```

## Engine 키

OpenClaw는 PREQ engine key를 로컬 CLI binary로 매핑합니다.

| Engine key | Local binary |
|---|---|
| `claude-code` | `claude` |
| `codex` | `codex` |
| `gemini-cli` | `gemini` |

기본 engine key: `claude-code`

선택한 CLI가 OpenClaw gateway process에서 접근 가능한 안정적인 실행 경로에 설치되어 있는지 확인하세요.

## Dispatch 명령

OpenClaw adapter는 일반 chat run 전에 명시적인 PREQ dispatch message를 가로채 처리합니다.

```text
!/skill preqstation-dispatch implement PROJ-284 using codex branch_name="task/proj-284-example"
/skill preqstation-dispatch review PROJ-284 using claude-code
/skill preqstation-dispatch ask PROJ-285 using gemini-cli ask_hint="Acceptance criteria"
```

웹 앱의 Telegram/OpenClaw send도 같은 OpenClaw command shape을 사용합니다.

## 다음 단계

1. plugin을 설치 또는 업데이트합니다
2. OpenClaw를 재시작합니다
3. `/preqsetup auto` 또는 `/preqsetup set`으로 project key를 매핑합니다
4. `claude`, `codex`, `gemini` 중 필요한 CLI가 gateway process에서 접근 가능한지 확인합니다
5. [실행](/guide/ko/openclaw/execution) 문서로 넘어갑니다

<Aside>
OpenClaw가 메시지, plugin config, 공유 매핑 파일, legacy fallback 어디에서도 프로젝트 경로를 못 찾으면 dispatch 전에 explicit mapping을 먼저 추가해야 합니다.
</Aside>
