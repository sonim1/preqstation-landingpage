이 페이지는 **OpenClaw dispatcher** 경로를 쓸 때 봅니다.

OpenClaw는 아래 경우에 적합합니다.

- Telegram 기반 dispatch를 쓰고 싶을 때
- 복사한 Telegram/OpenClaw 명령을 실행하고 싶을 때
- OpenClaw의 프로젝트 키/경로 해석이 필요할 때
- OpenClaw runtime이 worktree 생성과 실행을 맡아야 할 때

## ClawHub Import

저장소 URL:

```text
https://github.com/sonim1/preqstation-openclaw
```

## 프로젝트 경로 매핑

실제 프로젝트 매핑은 이 저장소가 아니라 **OpenClaw agent memory**에 저장됩니다.

`/Users/kendrick/projects/preqstation-openclaw/MEMORY.md`는 샘플 스키마 참고용일 뿐입니다.

예를 들어 짧은 소문자 키를 절대 경로에 매핑합니다.

```markdown
| key | cwd | note |
|-----|-----|------|
| proj | /Users/kendrick/projects/my-app | Main product repo |
| docs | /Users/kendrick/projects/preqstation-lp | Guide site |
```

해석 순서:

1. 메시지에 절대 경로가 직접 포함된 경우
2. OpenClaw agent memory의 프로젝트 키
3. OpenClaw agent memory의 task prefix 키 매칭
4. 없으면 유저에게 절대 경로를 물어보고 memory에 저장

## 환경 변수

### OPENCLAW_WORKTREE_ROOT

task worktree를 어디에 만들지 결정합니다.

```bash
export OPENCLAW_WORKTREE_ROOT="/tmp/openclaw-worktrees"
```

기본값: `/tmp/openclaw-worktrees`

## Worktree 레이아웃

각 task는 전용 worktree에서 실행됩니다.

```text
<OPENCLAW_WORKTREE_ROOT>/<project_key>/<branch_slug>/
```

브랜치 이름 우선순위:

1. 메시지의 `branch_name="..."`
2. fallback `preqstation/<project_key>`

custom branch 이름에 project key가 없다면 OpenClaw가 아래처럼 정규화합니다.

```text
preqstation/<project_key>/<branch_name>
```

예시:

```text
/tmp/openclaw-worktrees/proj/preqstation-proj-rate-limit/
```

## Engine 키

OpenClaw는 PREQ engine 키를 로컬 CLI 바이너리로 매핑해 실행합니다.

| Engine key | Local binary |
|---|---|
| `claude-code` | `claude` |
| `codex` | `codex` |
| `gemini-cli` | `gemini` |

기본 engine 키: `claude-code`

즉 OpenClaw 경로에서도 아래 엔진을 그대로 실행할 수 있습니다.

- Claude Code
- Codex
- Gemini CLI

자연어 예시:

- `Start PROJ-284 in the example project using Claude Code.`
- `Use Codex to fix README command examples in the example project.`
- `Use Gemini CLI to draft notes for DOC-12 in the example project.`

## 다음 단계

1. OpenClaw runtime에 skill을 import합니다
2. agent memory에 project key → path 매핑이 있는지 확인합니다
3. 기본 경로를 쓰기 싫다면 `OPENCLAW_WORKTREE_ROOT`를 설정합니다
4. `claude`, `codex`, `gemini` 중 필요한 CLI가 로컬에 설치되어 있는지 확인합니다
5. [실행](/guide/ko/openclaw/execution) 문서로 넘어갑니다

<Aside>
OpenClaw가 프로젝트 경로를 해석하지 못하면, 절대 경로를 물어본 뒤 agent memory에 저장해야 합니다.
</Aside>
