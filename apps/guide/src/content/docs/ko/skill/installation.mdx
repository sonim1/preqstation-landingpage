`preqstation-skill`은 PREQSTATION의 핵심 에이전트 측 패키지입니다. `preqstation` 스킬 이름을 소유하고, 작업 수명 주기 규칙을 문서화하며, 선택적으로 셸 헬퍼 fallback도 제공합니다.

Claude Code에서는 plugin install이 이제 기본 경로입니다. plugin helpers 없이 worker-only 모드만 원할 때만 standalone `preqstation` 스킬을 설치하세요.

## 사전 준비

- Node.js 18+ 설치
- `https://<your-domain>`에서 동작 중인 PREQSTATION 배포
- 권장 원격 MCP 로그인 플로우를 쓰려면 PREQSTATION 웹 앱 접근 권한
- 셸 헬퍼 모드를 쓸 계획이라면 PREQSTATION API 토큰

## 프로젝트 경로 매핑

`/preqstation:setup`은 Claude plugin과 worker 설정에서 쓰는 path mapping 진입점입니다.

이 명령은 다음을 수행합니다.

- MCP 서버가 지원할 때 `preq_list_projects`로 PREQ 프로젝트를 가져옵니다
- 로컬 매핑을 `~/.preqstation-dispatch/projects.json`에 저장합니다
- 자동 탐색이 로컬 repo를 찾기 어려우면 `PREQSTATION_REPO_ROOTS`를 fallback으로 사용합니다
- 자동 매칭이 안 되면 절대 경로를 수동으로 받을 수 있습니다

매핑 파일 형태는 다음과 같습니다.

```json
{
  "projects": {
    "PROJ": "/absolute/path/to/repo"
  }
}
```

## 스킬 설치 (에이전트별 권장)

### Claude Code

<Tabs>
<TabItem label="Plugin install (default)">
```bash
claude plugin marketplace add https://github.com/sonim1/preqstation-skill
claude plugin install preqstation@preqstation
claude plugin list
```

설치 후 다음을 실행하세요.

```text
/preqstation:setup
```

이렇게 하면 로컬 project mapping 파일이 설정되고 plugin helper가 worker 체크아웃과 같은 repo 경로를 보게 됩니다.
</TabItem>
<TabItem label="Worker-only mode">
```bash
npx skills add sonim1/preqstation-skill -g -a claude-code
```

plugin helpers 없이 핵심 worker 스킬만 쓰고 싶을 때만 이 경로를 사용하세요.
</TabItem>
</Tabs>

### Codex

```bash
npx skills add sonim1/preqstation-skill -g -a codex
```

### Gemini CLI

```bash
npx skills add sonim1/preqstation-skill -g -a gemini-cli
```

OpenClaw와 연결된 스킬이 공용 skills store에 노출되는 것을 원하지 않는다면 bare shared install은 피하세요.

PREQ 엔진 값은 아래 에이전트 타깃과 일치합니다.

```bash
claude-code
codex
gemini-cli
```

## Dispatcher 경로는 별도 문서로 분리

이 페이지는 **worker 설정 전용**입니다.

웹 앱에서 queued dispatch도 쓰고 싶다면 [OpenClaw 설치](/guide/ko/openclaw/setup)로 Telegram/OpenClaw dispatcher 경로를 추가하세요.

## 권장 MCP 설정 (HTTP + OAuth)

권장 MCP 런타임은 이제 PREQSTATION 서비스 내부의 `/mcp`에 있습니다. Claude Code, Codex, Gemini CLI는 이 원격 엔드포인트에 연결하고, 필요할 때 브라우저 OAuth 플로우를 완료하면 됩니다.

Claude Code plugin과 worker 설정에서는 `/mcp`를 등록한 뒤 `/preqstation:setup`을 실행하세요. installer가 repo를 자동 탐지하지 못하면 `PREQSTATION_REPO_ROOTS`를 설정하거나 절대 경로를 수동으로 넣으면 됩니다.

## 인증이 시작되는 시점

- Codex는 `add` 중에 서버를 미리 확인하는 경우가 많아서 브라우저 로그인 플로우가 즉시 시작될 수 있습니다.
- Gemini CLI도 설치된 CLI 버전에 따라 `mcp add` 시점이나 첫 실제 호출 시점에 인증이 시작될 수 있습니다.
- Claude Code는 보통 서버 설정을 먼저 저장하고, 실제 사용 전까지 `Needs authentication`으로 보일 수 있습니다.
- Claude Code에서는 처음 서버를 사용할 때나 `claude mcp get preqstation`을 실행할 때 인증이 시작되는 경우가 많습니다.

## Worker와 Dispatcher의 차이

- **worker 설정**은 에이전트가 MCP를 통해 PREQ 툴을 직접 호출하는 경로입니다.
- **dispatcher 설정**은 웹 앱이 다른 runtime에 작업을 큐잉하는 경로입니다.
- **engine**은 별도이며 `claude-code`, `codex`, `gemini-cli` 중 실제 실행 CLI를 결정합니다.

직접 worker만 쓸 거라면 이 페이지로 충분합니다. 웹 앱 queued dispatch가 필요하면 [OpenClaw 설치](/guide/ko/openclaw/setup)로 넘어가세요.

## Gemini CLI

Gemini CLI는 설치된 로컬 CLI가 remote HTTP MCP를 지원할 때 Claude/Codex와 같은 worker + remote MCP 경로를 사용합니다.

원격 PREQSTATION MCP 엔드포인트를 등록합니다.

<Tabs>
<TabItem label="설치">
```bash
gemini mcp add --scope user --transport http preqstation https://<your-domain>/mcp
```
</TabItem>
<TabItem label="확인">
```bash
gemini mcp list
```
</TabItem>
<TabItem label="실행">
```bash
gemini
# 또는
gemini -p "Read and execute the PREQSTATION task instructions from this workspace."
```
</TabItem>
</Tabs>

설치된 Gemini CLI가 remote HTTP MCP를 지원하지 않는다면 셸 헬퍼 fallback을 사용하세요.

```bash
export PREQSTATION_API_URL="https://your-preqstation-domain.vercel.app"
export PREQSTATION_TOKEN="preq_xxxxxxxxxxxxxxxxx"
export PREQSTATION_ENGINE="gemini-cli"  # optional fallback
```

## 셸 헬퍼 fallback (선택 사항)

MCP를 사용할 수 없을 때만 셸 헬퍼를 사용하세요.

먼저 아래 변수를 설정하세요.

```bash
export PREQSTATION_API_URL="https://your-preqstation-domain.vercel.app"
export PREQSTATION_TOKEN="preq_xxxxxxxxxxxxxxxxx"
export PREQSTATION_ENGINE="claude-code"  # or codex, gemini-cli
```

## 문제 해결

**원격 MCP에 `Needs authentication`이 표시될 때:**
- 서버 URL 끝이 `/mcp`인지 확인하세요.
- `claude mcp get preqstation`, `codex mcp get preqstation`, `gemini mcp list`로 첫 실제 연결이나 등록 상태를 확인하세요.
- 브라우저 로그인 플로우가 뜨면 완료하세요.

**셸 헬퍼에서 `401 Unauthorized`가 날 때:**
- 토큰 형식이 `preq_`로 시작하는지 확인
- `curl -H "Authorization: Bearer $PREQSTATION_TOKEN" "$PREQSTATION_API_URL/api/health"`로 연결 확인

**토큰이 거부될 때:**
- `/connections?legacy=1`의 레거시 토큰 플로우에서 새 토큰을 발급
- 토큰 만료 여부 확인
