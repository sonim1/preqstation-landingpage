`preqstation-skill`은 PREQSTATION의 핵심 에이전트 측 패키지입니다. `preqstation` 스킬 이름을 소유하고, 작업 수명 주기 규칙을 문서화하며, 선택적으로 셸 헬퍼와 레거시 stdio 브리지도 제공합니다.

## 사전 준비

- Node.js 22+ 설치
- `https://<your-domain>`에서 동작 중인 PREQSTATION 배포
- 권장 원격 MCP 로그인 플로우를 쓰려면 PREQSTATION 웹 앱 접근 권한
- 셸 헬퍼 모드나 레거시 로컬 stdio 브리지를 쓸 계획이라면 PREQSTATION API 토큰

## 스킬 설치 (에이전트별 권장)

핵심 `preqstation` 스킬을 소유해야 하는 에이전트에만 설치하세요.

```bash
npx skills add sonim1/preqstation-skill -g -a claude-code
npx skills add sonim1/preqstation-skill -g -a codex
npx skills add sonim1/preqstation-skill -g -a gemini-cli
```

OpenClaw와 연결된 스킬이 공용 skills store에 노출되는 것을 원하지 않는다면 bare shared install은 피하세요.

PREQ 엔진 값은 아래 에이전트 타깃과 일치합니다.

```bash
claude-code
codex
gemini-cli
```

## Dispatcher 경로는 별도 문서로 분리

이 페이지는 **worker 설정 전용**입니다.

웹 앱에서 queued dispatch도 쓰고 싶다면 dispatcher 경로를 별도로 고르세요.

- [Claude Code Dispatch 설치](/guide/ko/claude-dispatch/installation) — OpenClaw 없이 Claude Code가 직접 dispatcher channel을 호스팅
- [OpenClaw 설치](/guide/ko/openclaw/setup) — Telegram/OpenClaw 기반 dispatcher 경로

## 권장 MCP 설정 (HTTP + OAuth)

권장 MCP 런타임은 이제 PREQSTATION 서비스 내부의 `/mcp`에 있습니다. Claude Code와 Codex는 이 원격 엔드포인트에 연결하고, 필요할 때 브라우저 OAuth 플로우를 완료하면 됩니다.

## Claude Code

원격 PREQSTATION MCP 엔드포인트를 등록합니다.

<Tabs>
<TabItem label="설치">
```bash
claude mcp add --transport http preqstation https://<your-domain>/mcp
```
</TabItem>
<TabItem label="업데이트">
이미 등록되어 있다면 제거 후 다시 추가하세요.

```bash
claude mcp remove preqstation
```

그다음 위 설치 명령을 다시 실행합니다.
</TabItem>
<TabItem label="확인">
```bash
claude mcp list
claude mcp get preqstation
```
</TabItem>
</Tabs>

## Codex CLI

원격 PREQSTATION MCP 엔드포인트를 등록합니다.

<Tabs>

</TabItem>
<TabItem label="업데이트">
이미 등록되어 있다면 제거 후 다시 추가하세요.

그다음 위 설치 명령을 다시 실행합니다.

Codex는 일반 worker 사용만 한다면 Claude 스타일의 로컬 plugin이나 channel runtime이 필요하지 않습니다. 나중에 [Claude Code Dispatch](/guide/ko/claude-dispatch/installation)를 쓸 경우에는 dispatcher 머신에 Codex CLI가 설치되어 있어야 하고 UI에서 task engine을 `Codex`로 선택하면 됩니다.

## 인증이 시작되는 시점

- Codex는 `add` 중에 서버를 미리 확인하는 경우가 많아서 브라우저 로그인 플로우가 즉시 시작될 수 있습니다.
- Claude Code는 보통 서버 설정을 먼저 저장하고, 실제 사용 전까지 `Needs authentication`으로 보일 수 있습니다.
- Claude Code에서는 처음 서버를 사용할 때나 `claude mcp get preqstation`을 실행할 때 인증이 시작되는 경우가 많습니다.

## Worker와 Dispatcher의 차이

- **worker 설정**은 에이전트가 MCP를 통해 PREQ 툴을 직접 호출하는 경로입니다.
- **dispatcher 설정**은 웹 앱이 다른 runtime에 작업을 큐잉하는 경로입니다.
- **engine**은 별도이며 `claude-code`, `codex`, `gemini-cli` 중 실제 실행 CLI를 결정합니다.

직접 worker만 쓸 거라면 이 페이지로 충분합니다.

## Gemini CLI

Gemini CLI는 여기서 문서화한 원격 MCP 등록 플로우를 직접 사용하지 않습니다. `gemini-cli`용 `preqstation` 스킬을 설치한 뒤, 직접 REST 접근이 필요할 때 셸 헬퍼 모드를 사용하세요. 이후 Claude Code Dispatch나 OpenClaw dispatch를 쓰고 싶다면 dispatcher 머신에 `gemini` CLI도 설치되어 있어야 합니다.

```bash
export PREQSTATION_API_URL="https://your-preqstation-domain.vercel.app"
export PREQSTATION_TOKEN="preq_xxxxxxxxxxxxxxxxx"
export PREQSTATION_ENGINE="gemini-cli"  # optional fallback
```

## 셸 헬퍼 또는 레거시 로컬 브리지 (선택 사항)

MCP를 사용할 수 없을 때는 셸 헬퍼나 레거시 로컬 stdio 브리지를 계속 사용할 수 있습니다.

먼저 아래 변수를 설정하세요.

```bash
export PREQSTATION_API_URL="https://your-preqstation-domain.vercel.app"
export PREQSTATION_TOKEN="preq_xxxxxxxxxxxxxxxxx"
export PREQSTATION_ENGINE="claude-code"  # or codex, gemini-cli
```

## JSON 설정 예시

레거시 로컬 stdio 브리지를 수동으로 설정해야 한다면 아래와 같이 구성합니다.

```json
{
  "mcpServers": {
    "preqstation": {
      "command": "node",
      "args": ["/ABSOLUTE/PATH/TO/preqstation-skill/scripts/preqstation-mcp-server.mjs"],
      "env": {
        "PREQSTATION_API_URL": "https://your-preqstation-domain.vercel.app",
        "PREQSTATION_TOKEN": "preq_xxxxxxxxxxxxxxxxx"
      }
    }
  }
}
```

<Aside>
`/ABSOLUTE/PATH/TO/preqstation-skill`은 실제 설치 경로로 바꾸세요. 상대 경로는 동작하지 않습니다. 위의 권장 원격 `/mcp` 설정은 로컬 스크립트 경로가 필요하지 않습니다.
</Aside>

## 문제 해결

**원격 MCP에 `Needs authentication`이 표시될 때:**
- 서버 URL 끝이 `/mcp`인지 확인하세요.
- `claude mcp get preqstation` 또는 `codex mcp get preqstation`으로 첫 실제 연결을 트리거하세요.
- 브라우저 로그인 플로우가 뜨면 완료하세요.

**레거시 로컬 브리지가 시작되지 않을 때:**
- Node.js 설치 여부 확인: `node --version`
- 스크립트 경로가 절대 경로인지 확인
- `PREQSTATION_API_URL`이 `https://`로 시작하는지 확인

**셸 헬퍼 또는 레거시 브리지에서 `401 Unauthorized`가 날 때:**
- 토큰 형식이 `preq_`로 시작하는지 확인
- `curl -H "Authorization: Bearer $PREQSTATION_TOKEN" "$PREQSTATION_API_URL/api/health"`로 연결 확인

**토큰이 거부될 때:**
- PREQSTATION 웹 앱의 `/api-keys`에서 새 토큰을 발급
- 토큰 만료 여부 확인
