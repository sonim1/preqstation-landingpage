PREQSTATION is an execution system for developers using Claude, Codex, and Gemini in real projects.

## Recommended First Step

If you're setting up PREQSTATION for the first time, start with [Quick Start](/guide/getting-started/quick-start). It gets the web app running, walks you through login, and ends with your first API key. If you want the full local checklist instead of the shortest path, continue with [Web App Setup](/guide/web-app/setup).

## Choose Your Setup Path

### 1. Web App Setup

- [Quick Start](/guide/getting-started/quick-start) — fastest path to local app startup, login, and your first API key
- [Web App Setup](/guide/web-app/setup) — full local setup details, environment variables, and first-login flow

### 2. Worker Connection

- [Worker Installation](/guide/skill/installation) — connect Claude, Codex, or Gemini after the app is already running

### 3. Dispatcher Setup

Choose a dispatcher only if you want queued delivery from the web app.

- [Claude Code Dispatch Installation](/guide/claude-dispatch/installation) — queued dispatch without OpenClaw
- [OpenClaw Installation](/guide/openclaw/setup) — Telegram/OpenClaw delivery from the web app

## How PREQSTATION Fits Together

- **PREQSTATION web app** — Central web app with the Kanban board, REST API, dispatch action UI, Telegram delivery, and execution overlays
- **preqstation-skill** — Agent-side package for worker MCP access
- **Claude Code Dispatch** — Local Claude-hosted dispatcher channel used by the web app's `Send to Claude Code` action
- **preqstation-openclaw** — OpenClaw dispatcher path for Telegram or direct runtime messages, still responsible for worktree-based launch on that path

Users choose a **dispatcher target** in the web app, and that choice is separate from the task's **engine**. The dispatcher decides who picks the task up. The engine still decides whether the run launches `claude-code`, `codex`, or `gemini-cli`.

Fresh install: start with the web app path. App already running with an API key: move to worker connection. Need queued delivery: add one dispatcher path.

Use it when a task is already scoped and you want reliable execution, not when you need a generic issue backlog.

Create tasks. Agents plan, implement, and verify. You review the result and decide what reaches `done`.
